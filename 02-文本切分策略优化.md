## 概述

文本切分是RAG系统的基础环节，直接影响检索精度和生成质量。合理的切分策略需要在以下方面取得平衡：

- **语义完整性**：保持句子、段落的完整性
- **检索粒度**：足够细以便精确定位，足够粗以保留上下文
- **长度控制**：适配embedding模型的最大输入长度
- **重叠设计**：避免关键信息被切断

### 切分不当的影响

❌ **切分过大**：
- 包含过多无关信息，引入噪声
- Embedding被稀释，相似度降低
- LLM处理成本高

❌ **切分过小**：
- 丢失重要上下文
- 语义不完整
- 召回率下降

✅ **理想切分**：
- 每个chunk表达一个完整的语义单元
- 长度适中（200-500 tokens）
- 带有充分的重叠和元数据

## 基础切分方法

### 1. 固定长度切分

最简单的切分方式，按字符数或token数切分。

```python
def fixed_chunk(text, chunk_size=512, overlap=50):
    """
    固定长度切分
    
    Args:
        text: 输入文本
        chunk_size: 每个chunk的大小（字符数）
        overlap: 重叠大小（字符数）
    
    Returns:
        chunks列表
    """
    chunks = []
    start = 0
    
    while start < len(text):
        end = start + chunk_size
        chunk = text[start:end]
        
        # 如果不是最后一个chunk，尝试在边界处调整
        if end < len(text):
            # 找到最近的句子边界
            for i in range(min(50, len(chunk))):
                pos = len(chunk) - i
                if chunk[pos-1] in ['。', '！', '？', '\n']:
                    chunk = chunk[:pos]
                    break
        
        chunks.append(chunk)
        start += (chunk_size - overlap)
    
    return chunks
```

**优点**：
- 实现简单
- 可控性强
- 速度快

**缺点**：
- 可能切断句子或段落
- 不考虑语义边界
- 需要手动调整边界

**适用场景**：
- 快速原型开发
- 对精度要求不高的场景
- 作为baseline对比

### 2. 基于分隔符切分

按照自然分隔符（段落、句子）进行切分。

```python
def separator_chunk(text, separators=None, min_length=100, max_length=512):
    """
    基于分隔符的智能切分
    
    Args:
        text: 输入文本
        separators: 分隔符优先级列表
        min_length: 最小chunk长度
        max_length: 最大chunk长度
    
    Returns:
        chunks列表
    """
    if separators is None:
        separators = ['\n\n', '\n', '。', '！', '？', '；', '，']
    
    def split_by_separator(text, sep):
        """按指定分隔符切分"""
        if sep in text:
            parts = text.split(sep)
            # 保留分隔符
            result = []
            for i, part in enumerate(parts):
                if i > 0:
                    result.append(sep)
                if part.strip():
                    result.append(part)
            return result
        return [text]
    
    # 递归切分
    chunks = [text]
    
    for sep in separators:
        new_chunks = []
        for chunk in chunks:
            if len(chunk) > max_length:
                sub_chunks = split_by_separator(chunk, sep)
                new_chunks.extend(sub_chunks)
            else:
                new_chunks.append(chunk)
        chunks = new_chunks
        
        # 如果所有chunk都满足长度要求，提前退出
        if all(len(c) <= max_length for c in chunks):
            break
    
    # 合并过短的chunk
    chunks = merge_short_chunks(chunks, min_length)
    
    return chunks

def merge_short_chunks(chunks, min_length=100):
    """合并相邻的短chunk"""
    merged = []
    current = ""
    
    for chunk in chunks:
        if len(current) + len(chunk) < min_length:
            current += chunk
        else:
            if current:
                merged.append(current)
            current = chunk
    
    if current:
        merged.append(current)
    
    return merged
```

**优点**：
- 保持语义完整性
- 符合人类阅读习惯
- 实现相对简单

**缺点**：
- chunk长度不均匀
- 可能产生过长或过短的chunk
- 需要后处理

**适用场景**：
- 结构化较好的文档
- 对语义完整性要求高的场景

## 智能切分策略

### 1. 递归字符切分（⭐推荐）

LangChain的经典实现，综合了多种策略的优点。

```python
from typing import List, Optional

class RecursiveCharacterTextSplitter:
    """
    递归字符切分器
    
    工作原理：
    1. 尝试用第一个分隔符切分
    2. 如果片段仍过大，递归使用下一个分隔符
    3. 直到满足大小要求或无法再切分
    """
    
    def __init__(
        self,
        chunk_size: int = 512,
        chunk_overlap: int = 50,
        separators: Optional[List[str]] = None,
        keep_separator: bool = True,
        length_function=len
    ):
        self.chunk_size = chunk_size
        self.chunk_overlap = chunk_overlap
        self.keep_separator = keep_separator
        self.length_function = length_function
        
        # 默认分隔符（按优先级排序）
        if separators is None:
            self.separators = [
                "\n\n",      # 段落
                "\n",        # 换行
                "。",        # 句号
                "！",        # 感叹号
                "？",        # 问号
                "；",        # 分号
                "，",        # 逗号
                " ",         # 空格
                ""           # 字符级别（最后手段）
            ]
        else:
            self.separators = separators
    
    def split_text(self, text: str) -> List[str]:
        """切分文本"""
        return self._split_text(text, self.separators)
    
    def _split_text(self, text: str, separators: List[str]) -> List[str]:
        """递归切分"""
        final_chunks = []
        
        # 获取合适的分隔符
        separator = separators[-1]
        new_separators = []
        
        for i, s in enumerate(separators):
            if s == "":
                separator = s
                break
            if s in text:
                separator = s
                new_separators = separators[i+1:]
                break
        
        # 按分隔符切分
        splits = self._split_with_separator(text, separator)
        
        # 处理每个split
        good_splits = []
        
        for s in splits:
            if self.length_function(s) < self.chunk_size:
                good_splits.append(s)
            else:
                # 如果当前split仍然太长
                if good_splits:
                    # 先保存已累积的good_splits
                    merged = self._merge_splits(good_splits, separator)
                    final_chunks.extend(merged)
                    good_splits = []
                
                # 递归处理过长的split
                if not new_separators:
                    # 没有更多分隔符，强制切分
                    final_chunks.append(s[:self.chunk_size])
                else:
                    # 使用下一个分隔符继续切分
                    other_info = self._split_text(s, new_separators)
                    final_chunks.extend(other_info)
        
        # 处理剩余的good_splits
        if good_splits:
            merged = self._merge_splits(good_splits, separator)
            final_chunks.extend(merged)
        
        return final_chunks
    
    def _split_with_separator(self, text: str, separator: str) -> List[str]:
        """按分隔符切分，可选择保留分隔符"""
        if separator == "":
            return list(text)
        
        if self.keep_separator:
            # 保留分隔符
            splits = text.split(separator)
            result = []
            for i, s in enumerate(splits):
                if i > 0:
                    result.append(separator)
                if s:
                    result.append(s)
            return result
        else:
            return text.split(separator)
    
    def _merge_splits(self, splits: List[str], separator: str) -> List[str]:
        """合并split，考虑重叠"""
        if not splits:
            return []
        
        merged_chunks = []
        current_chunk = splits[0]
        
        for i in range(1, len(splits)):
            s = splits[i]
            
            # 检查加入下一个split是否会超过限制
            if self.length_function(current_chunk + s) <= self.chunk_size:
                current_chunk += s
            else:
                # 保存当前chunk
                merged_chunks.append(current_chunk)
                
                # 添加重叠部分
                if self.chunk_overlap > 0 and current_chunk:
                    overlap_start = max(0, len(current_chunk) - self.chunk_overlap)
                    current_chunk = current_chunk[overlap_start:] + s
                else:
                    current_chunk = s
        
        # 添加最后一个chunk
        if current_chunk:
            merged_chunks.append(current_chunk)
        
        return merged_chunks


# 使用示例
if __name__ == '__main__':
    splitter = RecursiveCharacterTextSplitter(
        chunk_size=512,
        chunk_overlap=50,
        separators=["\n\n", "\n", "。", "！", "？", "；", "，", " ", ""]
    )
    
    text = "这是第一段内容。\n\n这是第二段内容。包含多个句子。"
    chunks = splitter.split_text(text)
    
    for i, chunk in enumerate(chunks):
        print(f"Chunk {i+1} ({len(chunk)} chars):")
        print(chunk[:100] + "..." if len(chunk) > 100 else chunk)
        print()
```

**优势**：
- 自动适应不同文本结构
- 优先在语义边界切分
- 灵活可配置

**配置建议**：

| 文档类型 | chunk_size | chunk_overlap | 理由 |
|---------|-----------|---------------|------|
| 技术文档 | 512 | 50-80 | 代码和术语需要较大上下文 |
| 对话记录 | 300 | 60-90 | 对话轮次需要连续性 |
| 新闻报道 | 400 | 30-50 | 段落相对独立 |
| 法律条文 | 600 | 80-100 | 条款间引用频繁 |
| 产品手册 | 450 | 50-70 | 步骤说明需要完整 |

### 2. 语义感知切分

使用NLP模型识别语义边界，实现更智能的切分。

```python
import spacy
from typing import List

class SemanticChunker:
    """
    基于句法结构的语义切分器
    
    使用spaCy识别句子边界和语义单元
    """
    
    def __init__(self, model='zh_core_web_sm', max_chunk_size=512):
        try:
            self.nlp = spacy.load(model)
        except OSError:
            print(f"请安装spaCy模型: python -m spacy download {model}")
            raise
        
        self.max_chunk_size = max_chunk_size
    
    def chunk(self, text: str) -> List[str]:
        """
        语义感知切分
        
        Args:
            text: 输入文本
        
        Returns:
            语义完整的chunks列表
        """
        doc = self.nlp(text)
        
        chunks = []
        current_chunk = []
        current_length = 0
        
        for sent in doc.sents:
            sent_text = sent.text.strip()
            sent_length = len(sent_text)
            
            # 如果加入当前句子会超出限制
            if current_length + sent_length > self.max_chunk_size and current_chunk:
                # 保存当前chunk
                chunks.append("".join(current_chunk))
                
                # 开始新chunk
                current_chunk = [sent_text + "\n"]
                current_length = sent_length
            else:
                current_chunk.append(sent_text + "\n")
                current_length += sent_length
        
        # 添加最后一个chunk
        if current_chunk:
            chunks.append("".join(current_chunk))
        
        return chunks
    
    def chunk_with_similarity(self, text: str, threshold=0.8) -> List[str]:
        """
        基于句子相似度的切分
        
        将相似的句子组合在一起，形成语义连贯的chunk
        
        Args:
            text: 输入文本
            threshold: 相似度阈值，高于此值认为属于同一主题
        """
        doc = self.nlp(text)
        sentences = [sent.text.strip() for sent in doc.sents if sent.text.strip()]
        
        if not sentences:
            return []
        
        chunks = []
        current_chunk = [sentences[0]]
        
        for i in range(1, len(sentences)):
            prev_sent = sentences[i-1]
            curr_sent = sentences[i]
            
            # 计算句子相似度
            similarity = self._compute_similarity(prev_sent, curr_sent)
            
            if similarity >= threshold:
                # 相似度高，加入当前chunk
                current_chunk.append(curr_sent)
            else:
                # 相似度低，开始新chunk
                chunks.append("\n".join(current_chunk))
                current_chunk = [curr_sent]
        
        # 添加最后一个chunk
        if current_chunk:
            chunks.append("\n".join(current_chunk))
        
        return chunks
    
    def _compute_similarity(self, sent1: str, sent2: str) -> float:
        """计算两个句子的相似度"""
        doc1 = self.nlp(sent1)
        doc2 = self.nlp(sent2)
        return doc1.similarity(doc2)


# 使用示例
if __name__ == '__main__':
    chunker = SemanticChunker(max_chunk_size=512)
    
    text = """
    人工智能是计算机科学的一个分支。它试图理解智能的实质。
    并生产出一种新的能以人类智能相似的方式做出反应的智能机器。
    
    机器学习是人工智能的核心技术之一。它通过算法从数据中学习。
    深度学习是机器学习的一个子领域。它使用多层神经网络。
    """
    
    chunks = chunker.chunk(text)
    
    for i, chunk in enumerate(chunks):
        print(f"Chunk {i+1}:")
        print(chunk)
        print("-" * 50)
```

**优势**：
- 真正理解语义边界
- 保持主题连贯性
- 适合长文档

**劣势**：
- 依赖NLP模型，速度较慢
- 需要安装额外依赖
- 中文模型效果有限

**适用场景**：
- 学术论文
- 长篇报告
- 需要高度语义连贯的场景

### 3. 基于标题层级切分

适用于Markdown、HTML等结构化文档。

```python
import re
from typing import List, Dict

class HierarchicalChunker:
    """
    基于标题层级的切分器
    
    保留文档的层级结构，为每个chunk附加上下文
    """
    
    def __init__(self, max_chunk_size=1000):
        self.max_chunk_size = max_chunk_size
    
    def chunk_markdown(self, markdown_text: str) -> List[Dict]:
        """
        切分Markdown文档
        
        Returns:
            List[Dict]: 包含content和metadata的chunks
        """
        # 提取标题和内容
        pattern = r'(#{1,6}\s+.+?\n)(.*?)(?=#{1,6}\s+|$)'
        sections = re.findall(pattern, markdown_text, re.DOTALL)
        
        chunks = []
        hierarchy = []  # 维护标题层级栈
        
        for title, content in sections:
            # 解析标题层级
            level = title.count('#')
            title_text = title.strip().lstrip('#').strip()
            
            # 更新层级栈
            hierarchy = [h for h in hierarchy if h['level'] < level]
            hierarchy.append({'level': level, 'title': title_text})
            
            # 构建完整上下文
            context = self._build_hierarchy(hierarchy)
            
            # 如果内容过长，进一步切分
            full_content = context + content
            
            if len(full_content) > self.max_chunk_size:
                # 对内容进行二次切分
                sub_chunks = self._split_long_content(content, context)
                chunks.extend(sub_chunks)
            else:
                chunks.append({
                    'content': full_content,
                    'metadata': {
                        'title': title_text,
                        'level': level,
                        'hierarchy': [h['title'] for h in hierarchy],
                        'word_count': len(content)
                    }
                })
        
        return chunks
    
    def _build_hierarchy(self, hierarchy: List[Dict]) -> str:
        """构建层级上下文字符串"""
        if not hierarchy:
            return ""
        
        lines = []
        for h in hierarchy:
            indent = "  " * (h['level'] - 1)
            lines.append(f"{indent}{h['title']}")
        
        return "\n".join(lines) + "\n\n"
    
    def _split_long_content(self, content: str, context: str) -> List[Dict]:
        """切分过长的内容"""
        # 使用递归字符切分器
        from langchain.text_splitter import RecursiveCharacterTextSplitter
        
        splitter = RecursiveCharacterTextSplitter(
            chunk_size=self.max_chunk_size - len(context),
            chunk_overlap=50
        )
        
        sub_texts = splitter.split_text(content)
        
        chunks = []
        for i, sub_text in enumerate(sub_texts):
            chunks.append({
                'content': context + sub_text,
                'metadata': {
                    'sub_index': i,
                    'total_subs': len(sub_texts)
                }
            })
        
        return chunks
    
    def chunk_html(self, html_text: str) -> List[Dict]:
        """
        切分HTML文档
        
        使用BeautifulSoup解析HTML结构
        """
        from bs4 import BeautifulSoup
        
        soup = BeautifulSoup(html_text, 'html.parser')
        
        chunks = []
        current_section = []
        current_heading = ""
        
        # 遍历所有元素
        for element in soup.find_all(['h1', 'h2', 'h3', 'h4', 'h5', 'h6', 'p', 'div']):
            if element.name.startswith('h'):
                # 遇到新标题，保存之前的section
                if current_section:
                    chunks.append({
                        'content': "\n".join(current_section),
                        'metadata': {'heading': current_heading}
                    })
                
                # 开始新section
                current_heading = element.get_text()
                current_section = []
            else:
                text = element.get_text().strip()
                if text:
                    current_section.append(text)
        
        # 添加最后一个section
        if current_section:
            chunks.append({
                'content': "\n".join(current_section),
                'metadata': {'heading': current_heading}
            })
        
        return chunks


# 使用示例
if __name__ == '__main__':
    chunker = HierarchicalChunker()
    
    markdown = """
# 第一章：引言

这是引言部分的内容。介绍了整个文档的背景。

## 1.1 研究背景

详细的研究背景说明。包含多个段落。

## 1.2 研究目标

本研究的目标是...

# 第二章：方法论

## 2.1 数据收集

数据收集的方法...
"""
    
    chunks = chunker.chunk_markdown(markdown)
    
    for chunk in chunks:
        print(f"标题: {chunk['metadata']['title']}")
        print(f"层级: {chunk['metadata']['level']}")
        print(f"内容预览: {chunk['content'][:100]}...")
        print("=" * 60)
```

**优势**：
- 保留文档结构
- 提供丰富的上下文
- 便于溯源和导航

**适用场景**：
- 技术文档
- 产品手册
- 知识库文章
- API文档

### 4. 基于内容的自适应切分

根据内容密度和重要性动态调整切分粒度。

```python
class AdaptiveChunker:
    """
    自适应切分器
    
    根据内容特征动态调整chunk大小
    """
    
    def __init__(self, base_size=400, min_size=200, max_size=800):
        self.base_size = base_size
        self.min_size = min_size
        self.max_size = max_size
    
    def chunk(self, text: str) -> List[str]:
        """自适应切分"""
        # 分析内容特征
        features = self.analyze_content(text)
        
        # 根据特征调整chunk大小
        chunk_size = self.calculate_chunk_size(features)
        
        # 执行切分
        splitter = RecursiveCharacterTextSplitter(
            chunk_size=chunk_size,
            chunk_overlap=int(chunk_size * 0.1)
        )
        
        return splitter.split_text(text)
    
    def analyze_content(self, text: str) -> Dict:
        """分析内容特征"""
        import jieba
        
        words = jieba.lcut(text)
        
        features = {
            'length': len(text),
            'sentence_count': text.count('。') + text.count('！') + text.count('？'),
            'avg_sentence_length': len(text) / max(1, text.count('。')),
            'technical_density': self.count_technical_terms(words),
            'has_code': self.contains_code(text),
            'has_list': self.contains_list(text),
        }
        
        return features
    
    def calculate_chunk_size(self, features: Dict) -> int:
        """根据特征计算合适的chunk大小"""
        size = self.base_size
        
        # 技术密度高 → 增大chunk（需要更多上下文）
        if features['technical_density'] > 0.3:
            size += 100
        
        # 包含代码 → 增大chunk（保持代码块完整）
        if features['has_code']:
            size += 150
        
        # 句子很长 → 增大chunk
        if features['avg_sentence_length'] > 50:
            size += 50
        
        # 限制范围
        size = max(self.min_size, min(self.max_size, size))
        
        return size
    
    def count_technical_terms(self, words: List[str]) -> float:
        """计算技术术语密度"""
        # 简化的技术术语词典
        technical_terms = {
            'API', 'SDK', 'HTTP', 'JSON', '数据库', '服务器',
            '算法', '模型', '训练', '推理', '部署'
        }
        
        tech_count = sum(1 for w in words if w in technical_terms)
        return tech_count / max(1, len(words))
    
    def contains_code(self, text: str) -> bool:
        """检测是否包含代码"""
        code_indicators = ['def ', 'class ', 'import ', 'function', '=>', '{', '}']
        return any(indicator in text for indicator in code_indicators)
    
    def contains_list(self, text: str) -> bool:
        """检测是否包含列表"""
        return bool(re.search(r'^[\d\-\*•]', text, re.MULTILINE))
```

## 重叠策略优化

### 为什么需要重叠

```
无重叠的问题：

Chunk 1: "...这是重要的概念A。这个概念"
Chunk 2: "的概念A在实际应用中..."
                          ↑ 
                    关键信息被切断！

有重叠的解决方案：

Chunk 1: "...这是重要的概念A。这个概念的概念A在实际"
Chunk 2: "这个概念的概念A在实际应用中非常重要..."
         ↑___________________↑
              重叠区域确保完整性
```

### 重叠大小选择指南

| 文档类型 | 推荐重叠比例 | 绝对大小 | 原因 |
|---------|------------|---------|------|
| 技术文档 | 10-15% | 50-80 chars | 代码和术语需要上下文 |
| 对话记录 | 20-30% | 60-100 chars | 对话轮次强依赖前文 |
| 新闻报道 | 5-10% | 20-40 chars | 段落相对独立 |
| 法律条文 | 15-20% | 80-120 chars | 条款间频繁引用 |
| 产品手册 | 10-15% | 50-70 chars | 步骤说明需要连续 |

### 智能重叠实现

```python
class SmartOverlapStrategy:
    """智能重叠策略"""
    
    def __init__(self, base_overlap=50):
        self.base_overlap = base_overlap
    
    def calculate_overlap(self, prev_chunk: str, next_chunk: str, 
                         context: Dict = None) -> int:
        """
        动态计算重叠大小
        
        Args:
            prev_chunk: 前一个chunk
            next_chunk: 下一个chunk
            context: 上下文信息（文档类型、语言等）
        
        Returns:
            重叠字符数
        """
        overlap = self.base_overlap
        
        # 因素1：文档类型
        if context:
            doc_type = context.get('type', 'general')
            type_multipliers = {
                'technical': 1.5,
                'conversation': 2.0,
                'legal': 1.8,
                'news': 0.8,
            }
            overlap *= type_multipliers.get(doc_type, 1.0)
        
        # 因素2：检查是否有未完成的句子
        if prev_chunk and not self.ends_with_sentence_boundary(prev_chunk):
            overlap += 30  # 增加重叠以包含完整句子
        
        # 因素3：检查专有名词是否被切断
        if self.has_truncated_entity(prev_chunk, next_chunk):
            overlap += 40
        
        # 因素4：代码块完整性
        if self.has_incomplete_code_block(prev_chunk):
            overlap = max(overlap, 100)
        
        return int(overlap)
    
    def ends_with_sentence_boundary(self, text: str) -> bool:
        """检查文本是否以句子边界结束"""
        return text.rstrip().endswith(('。', '！', '？', '\n'))
    
    def has_truncated_entity(self, prev: str, next: str) -> bool:
        """检查是否有被切断的实体"""
        # 简化实现：检查是否有未闭合的引号、括号等
        open_chars = ['(', '[', '{', '"', ''']
        close_chars = [')', ']', '}', '"', ''']
        
        for open_c, close_c in zip(open_chars, close_chars):
            if prev.count(open_c) != prev.count(close_c):
                return True
        
        return False
    
    def has_incomplete_code_block(self, text: str) -> bool:
        """检查是否有未完成的代码块"""
        # 检查Markdown代码块
        code_block_count = text.count('```')
        return code_block_count % 2 != 0
```

### 重叠质量评估

```python
def evaluate_overlap_quality(chunks: List[str], original_text: str) -> Dict:
    """
    评估重叠策略的质量
    
    指标：
    1. 信息保留率：原始信息是否在chunks中完整保留
    2. 冗余度：重叠带来的冗余程度
    3. 边界完整性：句子/段落是否在边界处被切断
    """
    
    # 1. 信息保留率
    reconstructed = "".join(chunks)
    # 去除重叠部分的重复
    unique_chars = set(reconstructed)
    original_chars = set(original_text)
    retention_rate = len(unique_chars & original_chars) / len(original_chars)
    
    # 2. 冗余度
    total_length = sum(len(c) for c in chunks)
    redundancy = (total_length - len(original_text)) / len(original_text)
    
    # 3. 边界完整性
    boundary_violations = 0
    for chunk in chunks:
        if chunk and not chunk.rstrip().endswith(('。', '！', '？', '\n', ' ')):
            boundary_violations += 1
    
    boundary_integrity = 1 - (boundary_violations / len(chunks))
    
    return {
        'retention_rate': retention_rate,
        'redundancy': redundancy,
        'boundary_integrity': boundary_integrity,
        'score': (retention_rate * 0.4 + 
                 (1 - redundancy) * 0.3 + 
                 boundary_integrity * 0.3)
    }
```

def weighted_search(query_embedding, index, weights: Dict) -> List:
    """
    基于元数据的加权检索
    
    Args:
        weights: {
            'freshness': 0.2,      # 新鲜度权重
            'authority': 0.15,     # 权威性权重
            'relevance': 0.65      # 相关性权重
        }
    """
    results = index.search(query_embedding, top_k=50)
    
    scored_results = []
    for result in results:
        metadata = result['metadata']
        
        # 基础相关性分数
        relevance_score = result['score']
        
        # 新鲜度分数（越近越高）
        freshness_score = compute_freshness(metadata.get('created_at'))
        
        # 权威性分数
        authority_score = compute_authority(metadata.get('source'))
        
        # 加权总分
        total_score = (
            weights['relevance'] * relevance_score +
            weights['freshness'] * freshness_score +
            weights['authority'] * authority_score
        )
        
        result['final_score'] = total_score
        scored_results.append(result)
    
    # 按总分排序
    scored_results.sort(key=lambda x: x['final_score'], reverse=True)
    
    return scored_results[:20]

def compute_freshness(created_at: str) -> float:
    """计算新鲜度分数"""
    from datetime import datetime
    
    if not created_at:
        return 0.5
    
    doc_date = datetime.fromisoformat(created_at)
    days_diff = (datetime.now() - doc_date).days
    
    # 指数衰减：30天内为1，365天后接近0
    import math
    return math.exp(-days_diff / 180)

def compute_authority(source: str) -> float:
    """计算权威性分数"""
    authority_map = {
        'official_docs': 1.0,
        'technical_manual': 0.9,
        'blog_post': 0.6,
        'user_forum': 0.4,
    }
    
    # 简化的来源分类
    if 'official' in source:
        return authority_map['official_docs']
    elif 'manual' in source:
        return authority_map['technical_manual']
    else:
        return 0.5
```

## 多粒度索引

同时维护不同粒度的索引，支持分层检索。

### 索引层级设计

```
文档层级结构：

Document Level (整篇文档)
├── Section Level (章节)
│   ├── Paragraph Level (段落)
│   │   └── Sentence Level (句子)
```

### 实现方案

```python
class MultiGranularityIndex:
    """
    多粒度索引
    
    同时维护文档、章节、段落三个粒度的索引
    """
    
    def __init__(self, embedding_model):
        self.model = embedding_model
        
        # 不同粒度的索引
        self.document_index = {}  # doc_id -> embedding
        self.section_index = {}   # section_id -> embedding
        self.paragraph_index = {} # para_id -> embedding
        
        # 映射关系
        self.doc_to_sections = {}  # doc_id -> [section_ids]
        self.section_to_paragraphs = {}  # section_id -> [para_ids]
    
    def index_document(self, document: Dict):
        """
        索引文档及其所有粒度
        
        Args:
            document: {
                'doc_id': 'doc_001',
                'title': '文档标题',
                'content': '完整内容',
                'sections': [
                    {
                        'section_id': 'sec_001',
                        'title': '第一章',
                        'content': '章节内容',
                        'paragraphs': [
                            {'para_id': 'para_001', 'content': '段落1'},
                            ...
                        ]
                    },
                    ...
                ]
            }
        """
        doc_id = document['doc_id']
        
        # 1. 文档级别索引
        doc_embedding = self.model.encode([document['content']])[0]
        self.document_index[doc_id] = {
            'embedding': doc_embedding,
            'metadata': {
                'title': document['title'],
                'type': 'document'
            }
        }
        
        self.doc_to_sections[doc_id] = []
        
        # 2. 章节级别索引
        for section in document['sections']:
            section_id = section['section_id']
            section_embedding = self.model.encode([section['content']])[0]
            
            self.section_index[section_id] = {
                'embedding': section_embedding,
                'metadata': {
                    'title': section['title'],
                    'doc_id': doc_id,
                    'type': 'section'
                }
            }
            
            self.doc_to_sections[doc_id].append(section_id)
            self.section_to_paragraphs[section_id] = []
            
            # 3. 段落级别索引
            for para in section['paragraphs']:
                para_id = para['para_id']
                para_embedding = self.model.encode([para['content']])[0]
                
                self.paragraph_index[para_id] = {
                    'embedding': para_embedding,
                    'metadata': {
                        'doc_id': doc_id,
                        'section_id': section_id,
                        'type': 'paragraph'
                    }
                }
                
                self.section_to_paragraphs[section_id].append(para_id)
    
    def hierarchical_search(self, query: str, top_k=10) -> List[Dict]:
        """
        分层检索
        
        策略：
        1. 先用文档级别快速定位相关文档
        2. 在这些文档的相关章节中检索
        3. 最后在段落级别精确定位
        
        Returns:
            混合粒度的检索结果
        """
        query_embedding = self.model.encode([query])[0]
        
        # 第一层：文档级别检索（快速筛选）
        doc_scores = self._search_index(query_embedding, self.document_index)
        top_docs = [item['id'] for item in doc_scores[:5]]  # Top 5文档
        
        # 第二层：章节级别检索
        candidate_sections = []
        for doc_id in top_docs:
            section_ids = self.doc_to_sections.get(doc_id, [])
            for sec_id in section_ids:
                candidate_sections.append(sec_id)
        
        section_scores = self._search_index(
            query_embedding,
            {sid: self.section_index[sid] for sid in candidate_sections}
        )
        top_sections = [item['id'] for item in section_scores[:10]]  # Top 10章节
        
        # 第三层：段落级别检索
        candidate_paragraphs = []
        for sec_id in top_sections:
            para_ids = self.section_to_paragraphs.get(sec_id, [])
            for para_id in para_ids:
                candidate_paragraphs.append(para_id)
        
        paragraph_scores = self._search_index(
            query_embedding,
            {pid: self.paragraph_index[pid] for pid in candidate_paragraphs}
        )
        
        # 返回Top-K段落，附带上下文
        results = []
        for item in paragraph_scores[:top_k]:
            para_id = item['id']
            para_data = self.paragraph_index[para_id]
            
            # 获取父级上下文
            section_id = para_data['metadata']['section_id']
            doc_id = para_data['metadata']['doc_id']
            
            section_title = self.section_index[section_id]['metadata']['title']
            doc_title = self.document_index[doc_id]['metadata']['title']
            
            results.append({
                'id': para_id,
                'content': item['content'],
                'score': item['score'],
                'context': {
                    'document': doc_title,
                    'section': section_title,
                },
                'metadata': para_data['metadata']
            })
        
        return results
    
    def _search_index(self, query_embedding: np.ndarray, index: Dict) -> List[Dict]:
        """在指定索引中搜索"""
        from sklearn.metrics.pairwise import cosine_similarity
        
        scores = []
        for item_id, item_data in index.items():
            similarity = cosine_similarity(
                query_embedding.reshape(1, -1),
                item_data['embedding'].reshape(1, -1)
            )[0][0]
            
            scores.append({
                'id': item_id,
                'score': float(similarity),
                'content': item_data.get('content', '')
            })
        
        # 按分数排序
        scores.sort(key=lambda x: x['score'], reverse=True)
        
        return scores
```

        """
        评估chunk内部连贯性
        
        方法：计算chunk内相邻句子的平均相似度
        """
        import spacy
        nlp = spacy.load('zh_core_web_sm')
        
        coherence_scores = []
        
        for chunk in chunks:
            doc = nlp(chunk)
            sentences = list(doc.sents)
            
            if len(sentences) < 2:
                coherence_scores.append(1.0)
                continue
            
            similarities = []
            for i in range(len(sentences) - 1):
                sim = sentences[i].similarity(sentences[i+1])
                similarities.append(sim)
            
            avg_coherence = np.mean(similarities)
            coherence_scores.append(avg_coherence)
        
        return np.mean(coherence_scores)
    
    def evaluate_information_density(self, chunks: List[str]) -> float:
        """
        评估信息密度
        
        方法：计算chunk的有效信息量 / 长度
        """
        densities = []
        
        for chunk in chunks:
            # 去除停用词后的有效词数
            import jieba
            stop_words = {'的', '了', '在', '是', '我', '有', '和', '就'}
            words = [w for w in jieba.lcut(chunk) if w not in stop_words and w.strip()]
            
            density = len(words) / max(1, len(chunk))
            densities.append(density)
        
        return np.mean(densities)
    
    def evaluate_retrieval_effectiveness(
        self, 
        chunks: List[str], 
        queries: List[str],
        relevant_chunks: List[List[int]]
    ) -> Dict:
        """
        评估检索有效性
        
        Args:
            queries: 测试查询列表
            relevant_chunks: 每个查询的相关chunk索引列表
        
        Returns:
            评估指标
        """
        # 编码所有chunks
        chunk_embeddings = self.model.encode(chunks)
        
        metrics = {
            'recall_at_5': [],
            'recall_at_10': [],
            'mrr': []
        }
        
        for query, rel_indices in zip(queries, relevant_chunks):
            query_emb = self.model.encode([query])
            
            # 计算相似度
            similarities = cosine_similarity(query_emb, chunk_embeddings)[0]
            top_indices = np.argsort(similarities)[::-1]
            
            # Recall@K
            for k in [5, 10]:
                retrieved = set(top_indices[:k])
                relevant = set(rel_indices)
                recall = len(retrieved & relevant) / len(relevant)
                metrics[f'recall_at_{k}'].append(recall)
            
            # MRR
            for rank, idx in enumerate(top_indices, 1):
                if idx in rel_indices:
                    metrics['mrr'].append(1.0 / rank)
                    break
            else:
                metrics['mrr'].append(0)
        
        # 汇总
        return {
            'recall_at_5': np.mean(metrics['recall_at_5']),
            'recall_at_10': np.mean(metrics['recall_at_10']),
            'mrr': np.mean(metrics['mrr'])
        }
```

```python
def tune_chunking_parameters(
    documents: List[str],
    test_queries: List[str],
    param_grid: Dict
) -> Dict:
    """
    网格搜索最优切分参数
    
    Args:
        param_grid: {
            'chunk_size': [300, 400, 500, 600],
            'chunk_overlap': [30, 50, 70, 100],
            'separator_priority': [...]
        }
    """
    best_score = 0
    best_params = None
    results = []
    
    # 准备ground truth
    relevant_chunks = prepare_ground_truth(documents, test_queries)
    
    # 网格搜索
    for chunk_size in param_grid['chunk_size']:
        for chunk_overlap in param_grid['chunk_overlap']:
            # 切分
            splitter = RecursiveCharacterTextSplitter(
                chunk_size=chunk_size,
                chunk_overlap=chunk_overlap
            )
            
            all_chunks = []
            for doc in documents:
                chunks = splitter.split_text(doc)
                all_chunks.extend(chunks)
            
            # 评估
            evaluator = ChunkingEvaluator(embedding_model)
            metrics = evaluator.evaluate_retrieval_effectiveness(
                all_chunks, test_queries, relevant_chunks
            )
            
            # 综合分数
            score = (
                metrics['recall_at_10'] * 0.5 +
                metrics['mrr'] * 0.3 +

            )
            
            results.append({

                },

            })
            
            if score > best_score:
                best_score = score
                best_params = {

                }

## 最佳实践

### ✅ Do's

1. **优先使用递归字符切分**
   ```python
   splitter = RecursiveCharacterTextSplitter(
       chunk_size=512,
       chunk_overlap=50,
       separators=["\n\n", "\n", "。", "！", "？", "，", " "]
   )
   ```

2. **根据文档类型调整参数**
   - 技术文档：chunk_size=600, overlap=80
   - 对话记录：chunk_size=300, overlap=70
   - 新闻报道：chunk_size=400, overlap=30

3. **始终添加元数据**
   ```python
   chunk_metadata = {
       'source': document_path,
       'position': {'page': page_num, 'section': section_title},
       'created_at': timestamp,
       'tags': extracted_keywords
   }
   ```

4. **对结构化文档使用层级切分**
   - Markdown/HTML：按标题层级切分
   - PDF：结合OCR和布局分析

5. **实施重叠策略**
   - 至少10%的重叠
   - 在句子边界处调整重叠

6. **评估切分质量**
   - 定期运行离线评估
   - 监控Recall@K和MRR

7. **建立多粒度索引**
   - 文档、章节、段落三级
   - 支持分层检索

### ❌ Don'ts

1. **不要使用固定长度硬切分**
   ```python
   # 错误！
   chunks = [text[i:i+512] for i in range(0, len(text), 512)]
   ```

2. **不要忽略重叠**
   ```python
   # 错误！
   chunks = split_text(text, chunk_size=512, overlap=0)
   ```

3. **不要在切分前不清理文本**
   ```python
   # 应该先清理
   text = clean_html(text)
   text = normalize_whitespace(text)
   chunks = splitter.split_text(text)
   ```

4. **不要对所有文档使用相同策略**
   - 根据文档类型选择切分器
   - 技术文档 vs 对话记录 vs 新闻

5. **不要忘记更新索引**
   - 新增文档后及时索引
   - 删除文档后清理索引

## 常见问题

### Q1: chunk_size应该设多大？

**A**: 
- 通用场景：400-512字符
- 技术文档：500-700字符（代码需要更多上下文）
- 简短问答：200-300字符
- 长文档摘要：600-800字符

关键是看embedding模型的最大长度限制和实际效果。

### Q2: 重叠会不会导致大量冗余？

**A**: 
适度的重叠（10-15%）带来的收益远大于成本：
- 存储增加约10-15%
- 但Recall提升可达20-30%
- 可以通过去重进一步优化

### Q3: 如何处理代码文档？

**A**:
```python
# 特殊处理代码块
def chunk_code_document(text):
    # 1. 保护代码块不被切断
    code_blocks = extract_code_blocks(text)
    
    # 2. 对非代码部分正常切分
    non_code_parts = remove_code_blocks(text)
    chunks = splitter.split_text(non_code_parts)
    
    # 3. 将完整代码块作为独立chunk
    for code in code_blocks:
        chunks.append(code)
    
    return chunks
```

### Q4: 切分后如何验证质量？

**A**:
1. 人工抽查：随机查看10-20个chunks
2. 离线评估：计算Recall@K、MRR
3. 在线A/B测试：对比不同切分策略的用户满意度

### Q5: 是否需要为每种文档类型定制切分器？

**A**: 
初期可以用统一的递归字符切分器，随着数据积累：
- 针对高频文档类型定制
- 建立切分器注册表
- 根据文件扩展名自动选择
