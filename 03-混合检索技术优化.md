## 概述

单一检索方式难以应对所有场景：
- **向量检索**擅长语义理解，但精确匹配弱
- **关键词检索**擅长精确匹配，但无法理解语义

混合检索结合多种技术的优势，显著提升召回质量和覆盖度。

### 核心价值

✅ **提高召回率**：不同检索方式互补，减少遗漏  
✅ **增强鲁棒性**：对查询变化更稳定  
✅ **平衡精度**：综合多种信号，排序更准确  
✅ **适应多样场景**：同时处理开放式问答和精确查询

## 检索方式对比

### 主要检索技术

| 检索类型 | 原理 | 优势 | 劣势 | 适用场景 |
|---------|------|------|------|---------|
| **向量检索** | 余弦相似度 | 语义理解强，容错性好 | 精确匹配弱，可能遗漏专有名词 | 开放式问答，模糊查询，概念解释 |
| **BM25** | 词频统计 | 成熟快速，可解释 | 不考虑语义，同义词问题 | 传统搜索，关键词匹配 |
| **Elasticsearch** | 倒排索引+评分 | 功能丰富，支持复杂查询 | 配置复杂，资源消耗大 | 生产环境，需要高级搜索功能 |
| **全文检索** | 正则/模式匹配 | 支持复杂语法 | 索引体积大，速度慢 | 特定格式数据，代码搜索 |

### 详细对比分析

#### 1. 向量检索（Dense Retrieval）

```python
from sentence_transformers import SentenceTransformer
from sklearn.metrics.pairwise import cosine_similarity

class VectorRetriever:
    def __init__(self, model_name='BAAI/bge-large-zh-v1.5'):
        self.model = SentenceTransformer(model_name)
        self.documents = []
        self.embeddings = None
    
    def index(self, documents: List[str]):
        """建立向量索引"""
        self.documents = documents
        self.embeddings = self.model.encode(documents, normalize_embeddings=True)
    
    def search(self, query: str, top_k=10) -> List[Dict]:
        """向量检索"""
        query_emb = self.model.encode([query], normalize_embeddings=True)
        
        # 计算余弦相似度
        similarities = cosine_similarity(query_emb, self.embeddings)[0]
        
        # Top-K
        top_indices = np.argsort(similarities)[::-1][:top_k]
        
        results = []
        for idx in top_indices:
            results.append({
                'id': idx,
                'content': self.documents[idx],
                'score': float(similarities[idx])
            })
        
        return results
```

**优势**：
- ✅ 理解语义："AI"和"人工智能"能匹配
- ✅ 容错性强：拼写错误、表述差异仍能召回
- ✅ 适合长尾查询：即使没有精确关键词也能找到相关内容

**劣势**：
- ❌ 精确匹配弱：产品型号"iPhone 15 Pro"可能被稀释
- ❌ 专有名词问题：人名、地名、术语可能召回不准
- ❌ 数值不敏感："价格100元"和"价格200元"可能相似度高

**典型案例**：
```
查询："如何学习机器学习"
✅ 向量检索能召回："深度学习入门教程"、"神经网络基础"
❌ 但可能错过："TensorFlow实战第3章"（如果没有语义关联）
```

#### 2. BM25关键词检索

```python
from rank_bm25 import BM25Okapi
import jieba

class BM25Retriever:
    def __init__(self):
        self.bm25 = None
        self.documents = []
        self.tokenized_docs = []
    
    def index(self, documents: List[str]):
        """建立BM25索引"""
        self.documents = documents
        # 中文分词
        self.tokenized_docs = [jieba.lcut(doc) for doc in documents]
        self.bm25 = BM25Okapi(self.tokenized_docs)
    
    def search(self, query: str, top_k=10) -> List[Dict]:
        """BM25检索"""
        query_tokens = jieba.lcut(query)
        
        # 获取BM25分数
        scores = self.bm25.get_scores(query_tokens)
        
        # Top-K
        top_indices = np.argsort(scores)[::-1][:top_k]
        
        results = []
        for idx in top_indices:
            if scores[idx] > 0:  # 过滤无匹配的
                results.append({
                    'id': idx,
                    'content': self.documents[idx],
                    'score': float(scores[idx])
                })
        
        return results
```

**优势**：
- ✅ 精确匹配：专有名词、产品型号准确召回
- ✅ 可解释性强：知道是因为哪些词匹配
- ✅ 速度快：成熟的算法，优化良好

**劣势**：
- ❌ 无语义理解："AI"和"人工智能"无法匹配
- ❌ 同义词问题：需要手动维护同义词词典
- ❌ 词汇鸿沟：用户用语和文档用语不一致时效果差

**典型案例**：
```
查询："iPhone 15 Pro 价格"
✅ BM25能精确召回包含"iPhone 15 Pro"和"价格"的文档
❌ 但无法召回："苹果手机最新款多少钱"（即使语义相关）
```

```python
from elasticsearch import Elasticsearch

class ElasticRetriever:
    def __init__(self, host='localhost:9200'):
        self.es = Elasticsearch([host])
        self.index_name = 'documents'
    
    def hybrid_search(self, query: str, query_embedding: List[float], 
                     top_k=10) -> List[Dict]:
        """ES原生混合检索"""
        
        search_body = {
            "size": top_k,
            "query": {
                "script_score": {
                    "query": {
                        "bool": {
                            "should": [
                                {
                                    "match": {
                                        "content": {
                                            "query": query,
                                            "boost": 0.3
                                        }
                                    }
                                }
                            ]
                        }
                    },
                    "script": {
                        "source": """
                            double vector_score = cosineSimilarity(
                                params.query_vector, 
                                'embedding'
                            );
                            double bm25_score = _score;
                            return 0.7 * vector_score + 0.3 * log(bm25_score + 1);
                        """,
                        "params": {
                            "query_vector": query_embedding
                        }
                    }
                }
            }
        }
        
        response = self.es.search(
            index=self.index_name,
            body=search_body
        )
        
        return response['hits']['hits']
```

**优势**：
- ✅ 一站式解决方案：同时支持向量和关键词
- ✅ 功能强大：过滤、聚合、高亮等
- ✅ 可扩展：分布式部署，海量数据

**劣势**：
- ❌ 复杂度高：需要学习和维护ES
- ❌ 资源消耗：内存和存储需求大
- ❌ 调参困难：多个参数需要优化

## 混合检索架构

### 1. 并行混合检索（推荐）

```
用户查询
    ↓
┌───────────────┐
│  Query预处理   │
└───────────────┘
    ↓
    ├──────────────┬──────────────┐
    ↓              ↓              ↓
┌────────┐   ┌──────────┐  ┌──────────┐
│向量检索 │   │BM25检索  │  │ES检索    │
└────────┘   └──────────┘  └──────────┘
    ↓              ↓              ↓
    └──────────────┴──────────────┘
                   ↓
          ┌────────────────┐
          │  结果融合(RRF)  │
          └────────────────┘
                   ↓
          ┌────────────────┐
          │   重排序        │
          └────────────────┘
                   ↓
              最终结果
```

**实现代码**：

```python
class ParallelHybridRetriever:
    """
    并行混合检索器
    
    同时执行多种检索，然后融合结果
    """
    
    def __init__(self, vector_retriever, keyword_retriever, 
                 weights=None):
        self.vector_retriever = vector_retriever
        self.keyword_retriever = keyword_retriever
        
        # 融合权重
        self.weights = weights or {
            'vector': 0.7,
            'keyword': 0.3
        }
    
    def search(self, query: str, query_embedding: np.ndarray,
              top_k=10) -> List[Dict]:
        """
        并行混合检索
        
        Args:
            query: 用户查询
            query_embedding: 查询的向量表示
            top_k: 返回结果数量
        
        Returns:
            融合后的检索结果
        """
        # 并行执行两种检索
        vector_results = self.vector_retriever.search(
            query_embedding, 
            top_k=top_k * 2  # 扩大召回
        )
        
        keyword_results = self.keyword_retriever.search(
            query, 
            top_k=top_k * 2
        )
        
        # 结果融合
        fused_results = self.fuse_results(
            vector_results, 
            keyword_results,
            top_k=top_k
        )
        
        return fused_results
    
    def fuse_results(self, vector_results: List[Dict],
                    keyword_results: List[Dict],
                    top_k: int) -> List[Dict]:
        """
        融合两路检索结果
        
        使用RRF（倒数排名融合）算法
        """
        return reciprocal_rank_fusion(
            [vector_results, keyword_results],
            weights=[self.weights['vector'], self.weights['keyword']],
            top_k=top_k
        )


def reciprocal_rank_fusion(result_lists: List[List[Dict]], 
                          weights: List[float] = None,
                          top_k: int = 10,
                          k: int = 60) -> List[Dict]:
    """
    RRF融合算法
    
    公式：score(d) = Σ (weight_i / (k + rank_i))
    
    Args:
        result_lists: 多路检索结果列表
        weights: 各路结果的权重
        top_k: 返回Top-K结果
        k: RRF平滑参数，通常设为60
    
    Returns:
        融合后的排序结果
    """
    if weights is None:
        weights = [1.0] * len(result_lists)
    
    # 文档ID -> {score, item}
    fused_scores = {}
    
    for results, weight in zip(result_lists, weights):
        for rank, item in enumerate(results, 1):
            doc_id = item.get('id', item.get('doc_id'))
            
            # RRF分数
            rrf_score = weight / (k + rank)
            
            if doc_id not in fused_scores:
                fused_scores[doc_id] = {
                    'score': 0,
                    'item': item
                }
            
            fused_scores[doc_id]['score'] += rrf_score
    
    # 按融合分数排序
    sorted_results = sorted(
        fused_scores.values(),
        key=lambda x: x['score'],
        reverse=True
    )
    
    # 返回Top-K
    return [item['item'] for item in sorted_results[:top_k]]
```

**优势**：
- ✅ 充分利用各检索方式的优势
- ✅ 实现简单，易于调试
- ✅ 灵活扩展：可以轻松添加更多检索路

**劣势**：
- ❌ 需要多次检索，延迟略高
- ❌ 需要合理设置权重

### 2. 串行混合检索（级联）

```
用户查询
    ↓
┌──────────────┐
│ 向量检索      │ ← 第一层：扩大召回
│ (召回100条)  │
└──────────────┘
    ↓
┌──────────────┐
│ 关键词过滤    │ ← 第二层：精确过滤
│ (筛选到30条) │
└──────────────┘
    ↓
┌──────────────┐
│ 重排序        │ ← 第三层：精细排序
│ (精选10条)   │
└──────────────┘
    ↓
  最终结果
```

**实现代码**：

```python
class CascadeHybridRetriever:
    """
    串行混合检索器（级联）
    
    先向量召回，再关键词过滤/重排
    """
    
    def __init__(self, vector_retriever, keyword_retriever):
        self.vector_retriever = vector_retriever
        self.keyword_retriever = keyword_retriever
    
    def search(self, query: str, query_embedding: np.ndarray,
              top_k=10) -> List[Dict]:
        """
        级联混合检索
        
        策略：
        1. 向量检索扩大召回（100条）
        2. 关键词过滤/加权（筛选到30条）
        3. 返回Top-K
        """
        # 第一步：向量检索扩大召回
        candidates = self.vector_retriever.search(
            query_embedding, 
            top_k=top_k * 5  # 召回50条
        )
        
        # 第二步：关键词过滤和重新打分
        filtered_results = self.keyword_refine(
            query, 
            candidates,
            min_match_score=0.2
        )
        
        # 第三步：返回Top-K
        return filtered_results[:top_k]
    
    def keyword_refine(self, query: str, candidates: List[Dict],
                      min_match_score: float = 0.2) -> List[Dict]:
        """
        使用关键词对候选结果进行精炼
        
        Args:
            query: 原始查询
            candidates: 向量检索的候选结果
            min_match_score: 最小关键词匹配分数
        
        Returns:
            精炼后的结果
        """
        query_tokens = set(jieba.lcut(query))
        
        refined = []
        for candidate in candidates:
            doc_tokens = set(jieba.lcut(candidate['content']))
            
            # 计算关键词重叠度
            overlap = len(query_tokens & doc_tokens)
            match_score = overlap / len(query_tokens) if query_tokens else 0
            
            # 过滤低匹配度的
            if match_score >= min_match_score:
                # 融合向量分数和关键词分数
                candidate['refined_score'] = (
                    0.6 * candidate['score'] +  # 向量分数
                    0.4 * match_score            # 关键词分数
                )
                refined.append(candidate)
        
        # 按融合分数排序
        refined.sort(key=lambda x: x['refined_score'], reverse=True)
        
        return refined
```

**优势**：
- ✅ 减少不必要的计算：只对候选集做关键词处理
- ✅ 保证语义相关性：向量检索作为第一道筛选
- ✅ 提升精确度：关键词过滤去除语义相关但不精确的结果

**劣势**：
- ❌ 可能遗漏：如果向量检索没召回，后续也无法补救
- ❌ 依赖第一层质量：向量检索必须足够好

**适用场景**：
- 向量检索召回率高但精度低的场景
- 需要严格过滤的场景（如法律、医疗）

## 结果融合算法

### 1. 倒数排名融合（RRF）⭐ 行业标准

**原理**：

RRF通过考虑文档在各路结果中的排名来计算融合分数，避免了对不同检索方式分数的归一化问题。

**公式**：

```
RRF Score(d) = Σ (1 / (k + rank_i))
```

其中：
- `d` 是文档
- `rank_i` 是文档在第i路结果中的排名
- `k` 是平滑参数（通常设为60）

**为什么k=60**：
- k值影响早期排名和后期排名的权重
- k=60是经过大量实验得出的经验值
- 较小的k（如10）会更重视排名靠前的结果
- 较大的k（如100）会平缓排名差异

**完整实现**：

```python
def advanced_rrf_fusion(result_lists: List[List[Dict]],
                       weights: List[float] = None,
                       top_k: int = 10,
                       k: int = 60,
                       normalize_scores: bool = True) -> List[Dict]:
    """
    高级RRF融合
    
    支持：
    - 自定义权重
    - 分数归一化
    - 去重
    - 元数据合并
    """
    if weights is None:
        weights = [1.0] * len(result_lists)
    
    # 文档ID -> 融合信息
    doc_info = {}
    
    for list_idx, (results, weight) in enumerate(zip(result_lists, weights)):
        for rank, item in enumerate(results, 1):
            doc_id = item.get('id', item.get('doc_id'))
            
            if doc_id is None:
                continue
            
            # 计算RRF分数
            rrf_score = weight / (k + rank)
            
            if doc_id not in doc_info:
                doc_info[doc_id] = {
                    'rrf_score': 0,
                    'ranks': {},
                    'scores': {},
                    'item': item.copy()
                }
            
            doc_info[doc_id]['rrf_score'] += rrf_score
            doc_info[doc_id]['ranks'][list_idx] = rank
            doc_info[doc_id]['scores'][list_idx] = item.get('score', 0)
    
    # 计算最终分数并排序
    scored_docs = []
    for doc_id, info in doc_info.items():
        # 可以在这里添加额外的融合逻辑
        final_score = info['rrf_score']
        
        # 附加融合信息到结果
        info['item'].update({
            'final_score': final_score,
            'rrf_score': info['rrf_score'],
            'ranks': info['ranks'],
            'fusion_method': 'rrf'
        })
        
        scored_docs.append(info['item'])
    
    # 按最终分数排序
    scored_docs.sort(key=lambda x: x['final_score'], reverse=True)
    
    return scored_docs[:top_k]
```

### 2. 加权分数融合

需要对各路分数进行归一化后再加权。

```python
def weighted_score_fusion(result_lists: List[List[Dict]],
                         weights: List[float] = None,
                         top_k: int = 10,
                         normalization='min_max') -> List[Dict]:
    """
    加权分数融合
    
    Args:
        normalization: 归一化方法 ('min_max', 'z_score', 'softmax')
    """
    if weights is None:
        weights = [1.0] * len(result_lists)
    
    # 归一化各路分数
    normalized_lists = []
    for results in result_lists:
        normalized = normalize_scores(results, method=normalization)
        normalized_lists.append(normalized)
    
    # 构建文档ID到结果的映射
    all_docs = {}
    
    for list_idx, (results, weight) in enumerate(zip(normalized_lists, weights)):
        for item in results:
            doc_id = item.get('id', item.get('doc_id'))
            
            if doc_id not in all_docs:
                all_docs[doc_id] = {
                    'scores': [],
                    'weights': [],
                    'item': item.copy()
                }
            
            all_docs[doc_id]['scores'].append(item.get('normalized_score', 0))
            all_docs[doc_id]['weights'].append(weight)
    
    # 计算加权分数
    scored_docs = []
    for doc_id, info in all_docs.items():
        # 加权平均
        weighted_score = sum(
            s * w for s, w in zip(info['scores'], info['weights'])
        ) / sum(info['weights'])
        
        info['item']['final_score'] = weighted_score
        scored_docs.append(info['item'])
    
    # 排序
    scored_docs.sort(key=lambda x: x['final_score'], reverse=True)
    
    return scored_docs[:top_k]


def normalize_scores(results: List[Dict], method='min_max') -> List[Dict]:
    """
    归一化分数到[0,1]区间
    
    Args:
        method: 归一化方法
            - 'min_max': Min-Max归一化
            - 'z_score': Z-Score标准化
            - 'softmax': Softmax归一化
    """
    if not results:
        return results
    
    scores = [item.get('score', 0) for item in results]
    
    if method == 'min_max':
        min_score = min(scores)
        max_score = max(scores)
        range_score = max_score - min_score
        
        if range_score == 0:
            normalized = [0.5] * len(scores)
        else:
            normalized = [(s - min_score) / range_score for s in scores]
    
    elif method == 'z_score':
        mean = np.mean(scores)
        std = np.std(scores)
        
        if std == 0:
            normalized = [0.5] * len(scores)
        else:
            normalized = [(s - mean) / std for s in scores]
            # 缩放到[0,1]
            min_n = min(normalized)
            max_n = max(normalized)
            normalized = [(n - min_n) / (max_n - min_n) for n in normalized]
    
    elif method == 'softmax':
        exp_scores = np.exp(scores)
        normalized = exp_scores / exp_scores.sum()
    
    else:
        raise ValueError(f"Unknown normalization method: {method}")
    
    # 附加归一化分数
    for item, norm_score in zip(results, normalized):
        item['normalized_score'] = norm_score
    
    return results
```

**适用场景**：
- 各路检索的分数分布已知且稳定
- 需要精细控制各路权重的场景

### 3. 学习排序融合（Learning to Rank）

使用机器学习模型自动学习最优融合策略。

```python
from sklearn.ensemble import GradientBoostingRegressor
import numpy as np

class LearningToRankFusion:
    """
    基于机器学习的结果融合
    
    使用标注数据训练融合模型，自动学习最优权重
    """
    
    def __init__(self):
        self.model = GradientBoostingRegressor(
            n_estimators=100,
            max_depth=5,
            learning_rate=0.1
        )
        self.is_trained = False
    
    def extract_features(self, vector_result: Dict, 
                        keyword_result: Dict,
                        query: str) -> np.ndarray:
        """
        提取融合特征
        
        特征工程是关键！
        """
        features = []
        
        # 1. 原始分数
        features.append(vector_result.get('score', 0))
        features.append(keyword_result.get('score', 0))
        
        # 2. 排名信息
        features.append(vector_result.get('rank', 0))
        features.append(keyword_result.get('rank', 0))
        
        # 3. 内容特征
        features.append(len(vector_result.get('content', '')))
        features.append(len(keyword_result.get('content', '')))
        
        # 4. 查询-文档匹配特征
        query_tokens = set(jieba.lcut(query))
        doc_tokens = set(jieba.lcut(vector_result.get('content', '')))
        
        # 词重叠率
        overlap = len(query_tokens & doc_tokens)
        overlap_rate = overlap / len(query_tokens) if query_tokens else 0
        features.append(overlap_rate)
        
        # 5. 元数据特征
        features.append(vector_result.get('metadata', {}).get('freshness', 0.5))
        features.append(vector_result.get('metadata', {}).get('authority', 0.5))
        
        return np.array(features)
    
    def train(self, training_data: List[Tuple]):
        """
        训练融合模型
        
        Args:
            training_data: [(query, results, relevance_scores), ...]
                - query: 查询字符串
                - results: 候选结果列表，每个结果包含vector_part和keyword_part
                - relevance_scores: 人工标注的相关性分数（0-5）
        """
        X = []
        y = []
        
        for query, results, relevance_scores in training_data:
            for result, relevance in zip(results, relevance_scores):
                features = self.extract_features(
                    result['vector_part'],
                    result['keyword_part'],
                    query
                )
                X.append(features)
                y.append(relevance)
        
        X = np.array(X)
        y = np.array(y)
        
        # 训练
        self.model.fit(X, y)
        self.is_trained = True
        
        print(f"模型训练完成，训练样本数: {len(X)}")
    
    def predict_relevance(self, vector_result: Dict,
                         keyword_result: Dict,
                         query: str) -> float:
        """预测相关性分数"""
        if not self.is_trained:
            raise Exception("Model not trained yet!")
        
        features = self.extract_features(
            vector_result,
            keyword_result,
            query
        ).reshape(1, -1)
        
        return self.model.predict(features)[0]
    
    def fuse(self, query: str,
            vector_results: List[Dict],
            keyword_results: List[Dict],
            top_k: int = 10) -> List[Dict]:
        """
        使用训练好的模型融合结果
        """
        if not self.is_trained:
            raise Exception("Model not trained yet!")
        
        # 构建文档ID映射
        vector_map = {r['id']: r for r in vector_results}
        keyword_map = {r['id']: r for r in keyword_results}
        
        all_doc_ids = set(vector_map.keys()) | set(keyword_map.keys())
        
        scored_docs = []
        for doc_id in all_doc_ids:
            vector_result = vector_map.get(doc_id, {
                'score': 0, 'rank': 999, 'content': ''
            })
            keyword_result = keyword_map.get(doc_id, {
                'score': 0, 'rank': 999, 'content': ''
            })
            
            # 预测相关性
            relevance_score = self.predict_relevance(
                vector_result,
                keyword_result,
                query
            )
            
            # 选择主要内容（优先使用向量检索的结果）
            content = vector_result.get('content', keyword_result.get('content', ''))
            
            scored_docs.append({
                'id': doc_id,
                'content': content,
                'final_score': relevance_score,
                'vector_score': vector_result.get('score', 0),
                'keyword_score': keyword_result.get('score', 0)
            })
        
        # 排序
        scored_docs.sort(key=lambda x: x['final_score'], reverse=True)
        
        return scored_docs[:top_k]
```

```python
from elasticsearch import Elasticsearch
from typing import List, Dict

class ElasticHybridSearch:
    """
    Elasticsearch混合检索完整实现
    
    支持：
    - 向量检索（dense_vector）
    - BM25关键词检索
    - 元数据过滤
    - 结果融合
    """
    
    def __init__(self, hosts=['localhost:9200'], index_name='documents'):
        self.es = Elasticsearch(hosts)
        self.index_name = index_name
    
    def create_index(self, embedding_dim=1024):
        """
        创建支持混合检索的索引
        
        Args:
            embedding_dim: 向量维度
        """
        mapping = {
            "settings": {
                "number_of_shards": 1,
                "number_of_replicas": 0,
                "analysis": {
                    "analyzer": {
                        "chinese_analyzer": {
                            "type": "custom",
                            "tokenizer": "ik_max_word"  # 中文分词
                        }
                    }
                }
            },
            "mappings": {
                "properties": {
                    "content": {
                        "type": "text",
                        "analyzer": "chinese_analyzer",
                        "fields": {
                            "keyword": {
                                "type": "keyword"
                            }
                        }
                    },
                    "title": {
                        "type": "text",
                        "analyzer": "chinese_analyzer",
                        "boost": 2.0  # 标题权重更高
                    },
                    "embedding": {
                        "type": "dense_vector",
                        "dims": embedding_dim,
                        "index": True,
                        "similarity": "cosine"  # 余弦相似度
                    },
                    "metadata": {
                        "properties": {
                            "source": {"type": "keyword"},
                            "category": {"type": "keyword"},
                            "created_at": {"type": "date"},
                            "author": {"type": "keyword"},
                            "tags": {"type": "keyword"}
                        }
                    }
                }
            }
        }
        
        # 删除旧索引（如果存在）
        if self.es.indices.exists(index=self.index_name):
            self.es.indices.delete(index=self.index_name)
        
        # 创建新索引
        self.es.indices.create(index=self.index_name, body=mapping)
        
        print(f"索引 '{self.index_name}' 创建成功")
    
    def index_document(self, doc_id: str, content: str,
                      embedding: List[float], metadata: Dict = None):
        """
        索引单个文档
        
        Args:
            doc_id: 文档ID
            content: 文档内容
            embedding: 向量表示
            metadata: 元数据
        """
        document = {
            "content": content,
            "embedding": embedding,
            "metadata": metadata or {}
        }
        
        self.es.index(
            index=self.index_name,
            id=doc_id,
            body=document
        )
    
    def bulk_index(self, documents: List[Dict]):
        """
        批量索引文档
        
        Args:
            documents: [
                {
                    'id': 'doc_001',
                    'content': '...',
                    'embedding': [...],
                    'metadata': {...}
                },
                ...
            ]
        """
        from elasticsearch.helpers import bulk
        
        actions = []
        for doc in documents:
            action = {
                "_index": self.index_name,
                "_id": doc['id'],
                "_source": {
                    "content": doc['content'],
                    "embedding": doc['embedding'],
                    "metadata": doc.get('metadata', {})
                }
            }
            actions.append(action)
        
        success, failed = bulk(self.es, actions)
        
        print(f"批量索引完成：成功{success}条，失败{len(failed)}条")
        
        if failed:
            print("失败的文档:", failed)
    
    def hybrid_search(self, query: str, query_embedding: List[float],
                     top_k: int = 10, filters: Dict = None) -> List[Dict]:
        """
        ES原生混合检索
        
        使用script_score结合向量相似度和BM25
        
        Args:
            query: 查询文本
            query_embedding: 查询向量
            top_k: 返回结果数
            filters: 过滤条件，如 {'category': 'technical'}
        
        Returns:
            检索结果列表
        """
        
        # 构建过滤条件
        filter_clauses = []
        if filters:
            for key, value in filters.items():
                filter_clauses.append({
                    "term": {
                        f"metadata.{key}": value
                    }
                })
        
        # 混合检索查询
        search_body = {
            "size": top_k,
            "query": {
                "script_score": {
                    "query": {
                        "bool": {
                            "must": [
                                {
                                    "match": {
                                        "content": {
                                            "query": query,
                                            "boost": 0.3
                                        }
                                    }
                                }
                            ],
                            "filter": filter_clauses  # 应用过滤
                        }
                    },
                    "script": {
                        "source": """
                            // 向量相似度分数
                            double vector_score = cosineSimilarity(
                                params.query_vector, 
                                'embedding'
                            );
                            
                            // BM25分数（取对数避免过大）
                            double bm25_score = _score;
                            
                            // 融合：70%向量 + 30%关键词
                            double final_score = 0.7 * vector_score + 0.3 * log(bm25_score + 1);
                            
                            return final_score;
                        """,
                        "params": {
                            "query_vector": query_embedding
                        }
                    }
                }
            },
            "_source": ["content", "metadata"]
        }
        
        # 执行搜索
        response = self.es.search(
            index=self.index_name,
            body=search_body
        )
        
        # 解析结果
        results = []
        for hit in response['hits']['hits']:
            results.append({
                'id': hit['_id'],
                'content': hit['_source']['content'],
                'score': hit['_score'],
                'metadata': hit['_source'].get('metadata', {})
            })
        
        return results
    
    def two_stage_search(self, query: str, query_embedding: List[float],
                        top_k: int = 10) -> List[Dict]:
        """
        两阶段搜索
        
        第一阶段：向量检索召回50条
        第二阶段：在召回结果中用BM25重排序
        
        适用于大规模索引，提高效率
        """
        
        # 第一阶段：向量检索
        vector_search_body = {
            "size": 50,
            "query": {
                "script_score": {
                    "query": {"match_all": {}},
                    "script": {
                        "source": "cosineSimilarity(params.query_vector, 'embedding') + 1.0",
                        "params": {
                            "query_vector": query_embedding
                        }
                    }
                }
            }
        }
        
        vector_response = self.es.search(
            index=self.index_name,
            body=vector_search_body
        )
        
        # 获取候选文档IDs
        candidate_ids = [hit['_id'] for hit in vector_response['hits']['hits']]
        
        if not candidate_ids:
            return []
        
        # 第二阶段：BM25重排序
        bm25_search_body = {
            "size": top_k,
            "query": {
                "bool": {
                    "must": [
                        {
                            "match": {
                                "content": query
                            }
                        }
                    ],
                    "filter": [
                        {
                            "ids": {
                                "values": candidate_ids
                            }
                        }
                    ]
                }
            }
        }
        
        bm25_response = self.es.search(
            index=self.index_name,
            body=bm25_search_body
        )
        
        # 解析结果
        results = []
        for hit in bm25_response['hits']['hits']:
            results.append({
                'id': hit['_id'],
                'content': hit['_source']['content'],
                'score': hit['_score'],
                'metadata': hit['_source'].get('metadata', {})
            })
        
        return results
    
    def delete_document(self, doc_id: str):
        """删除文档"""
        self.es.delete(index=self.index_name, id=doc_id)
    
    def update_document(self, doc_id: str, updates: Dict):
        """更新文档"""
        self.es.update(index=self.index_name, id=doc_id, body={"doc": updates})
    
    def get_stats(self) -> Dict:
        """获取索引统计信息"""
        stats = self.es.cat.indices(index=self.index_name, format='json')
        return stats[0] if stats else {}
```

## 动态权重调整

根据查询特征和上下文动态调整各路检索的权重。

### 基于规则的动态权重

```python
class DynamicWeightAdjuster:
    """
    动态权重调整器
    
    根据查询特征自动调整向量检索和关键词检索的权重
    """
    
    def __init__(self):
        # 默认权重
        self.default_weights = {
            'vector': 0.7,
            'keyword': 0.3
        }
    
    def adjust_weights(self, query: str, context: Dict = None) -> Dict:
        """
        动态调整权重
        
        Args:
            query: 用户查询
            context: 上下文信息（用户历史、时间等）
        
        Returns:
            调整后的权重
        """
        weights = self.default_weights.copy()
        
        # 规则1：包含引号 → 提高关键词权重
        if '"' in query or '"' in query:
            weights['vector'] -= 0.2
            weights['keyword'] += 0.2
        
        # 规则2：包含专有名词（大写字母+数字）→ 提高关键词权重
        if re.search(r'[A-Z]+\d+', query):
            weights['vector'] -= 0.15
            weights['keyword'] += 0.15
        
        # 规则3：长问句 → 提高向量权重
        if len(query) > 30 and ('?' in query or '？' in query):
            weights['vector'] += 0.1
            weights['keyword'] -= 0.1
        
        # 规则4：包含技术术语 → 平衡权重
        technical_terms = ['API', 'SDK', 'HTTP', 'JSON', '数据库']
        if any(term in query for term in technical_terms):
            weights['vector'] = 0.5
            weights['keyword'] = 0.5
        
        # 规则5：基于用户历史（如果有）
        if context and 'user_history' in context:
            user_pref = self.learn_from_history(context['user_history'])
            weights['vector'] = user_pref.get('vector_weight', weights['vector'])
            weights['keyword'] = 1 - weights['vector']
        
        # 确保权重和为1且在合理范围内
        weights['vector'] = max(0.3, min(0.9, weights['vector']))
        weights['keyword'] = 1 - weights['vector']
        
        return weights
    
    def learn_from_history(self, user_history: List[Dict]) -> Dict:
        """从用户历史中学习偏好"""
        # 简化实现：统计用户点击的文档来源
        vector_clicks = 0
        keyword_clicks = 0
        
        for interaction in user_history[-20:]:  # 最近20次交互
            if interaction.get('clicked_source') == 'vector':
                vector_clicks += 1
            elif interaction.get('clicked_source') == 'keyword':
                keyword_clicks += 1
        
        total = vector_clicks + keyword_clicks
        if total == 0:
            return {'vector_weight': 0.7}
        
        return {
            'vector_weight': vector_clicks / total
        }
```

```python
        self.model = RandomForestRegressor(n_estimators=50)
        self.is_trained = False
    
    def extract_query_features(self, query: str) -> np.ndarray:
        """提取查询特征"""
        features = [
            len(query),                              # 查询长度
            query.count(' '),                        # 词数近似
            1 if '?' in query or '？' in query else 0,  # 是否问句
            1 if '"' in query else 0,                # 是否有引号
            len(re.findall(r'\d+', query)),          # 数字数量
            len(re.findall(r'[A-Z]', query)),        # 大写字母数
            len(jieba.lcut(query)),                  # 分词数
        ]
        
        return np.array(features)
    
    def train(self, training_data: List[Tuple]):
        """
        训练权重预测模型
        
        Args:
            training_data: [(query, optimal_weights), ...]
                optimal_weights需要通过A/B测试或人工标注获得
        """
        X = []
        y = []
        
        for query, optimal_weights in training_data:
            features = self.extract_query_features(query)
            X.append(features)
            y.append(optimal_weights['vector'])  # 只需预测向量权重
        
        X = np.array(X)
        y = np.array(y)
        
        self.model.fit(X, y)
        self.is_trained = True
    
    def predict_weights(self, query: str) -> Dict:
        """预测最优权重"""
        if not self.is_trained:
            raise Exception("Model not trained!")
        
        features = self.extract_query_features(query).reshape(1, -1)
        vector_weight = self.model.predict(features)[0]
        
        # 限制范围
        vector_weight = max(0.3, min(0.9, vector_weight))
        
        return {
            'vector': vector_weight,
            'keyword': 1 - vector_weight
        }
```

## 评估与优化

### 离线评估指标

```python
class HybridSearchEvaluator:
    """混合检索评估器"""
    
    def __init__(self):
        pass
    
    def evaluate_recall(self, retriever, test_queries: List[str],
                       relevant_docs: List[List[str]],
                       top_k: int = 10) -> float:
        """
        评估召回率
        
        Args:
            retriever: 检索器
            test_queries: 测试查询列表
            relevant_docs: 每个查询的相关文档ID列表
            top_k: 召回数量
        
        Returns:
            平均召回率
        """
        recall_scores = []
        
        for query, rel_docs in zip(test_queries, relevant_docs):
            # 执行检索
            results = retriever.search(query, top_k=top_k)
            retrieved_ids = set(r['id'] for r in results)
            
            # 计算召回率
            rel_set = set(rel_docs)
            if rel_set:
                recall = len(retrieved_ids & rel_set) / len(rel_set)
                recall_scores.append(recall)
        
        return np.mean(recall_scores) if recall_scores else 0.0
    
    def evaluate_ndcg(self, retriever, test_queries: List[str],
                     relevant_docs: List[Dict[str, float]],
                     top_k: int = 10) -> float:
        """
        评估NDCG
        
        Args:
            relevant_docs: [{doc_id: relevance_score}, ...]
        """
        ndcg_scores = []
        
        for query, rel_dict in zip(test_queries, relevant_docs):
            results = retriever.search(query, top_k=top_k)
            
            # 计算DCG
            dcg = 0.0
            for i, result in enumerate(results):
                rel = rel_dict.get(result['id'], 0.0)
                dcg += rel / np.log2(i + 2)
            
            # 计算理想DCG
            ideal_rels = sorted(rel_dict.values(), reverse=True)[:top_k]
            idcg = sum(rel / np.log2(i + 2) for i, rel in enumerate(ideal_rels))
            
            ndcg = dcg / idcg if idcg > 0 else 0.0
            ndcg_scores.append(ndcg)
        
        return np.mean(ndcg_scores)
    
    def compare_strategies(self, retrievers: Dict, test_data: Dict) -> Dict:
        """
        对比不同检索策略
        
        Args:
            retrievers: {'strategy_name': retriever, ...}
            test_data: {'queries': [...], 'relevant_docs': [...]}
        """
        results = {}
        
        for name, retriever in retrievers.items():
            recall = self.evaluate_recall(
                retriever,
                test_data['queries'],
                test_data['relevant_docs']
            )
            
            results[name] = {
                'recall@10': recall,
            }
        
        return results
```

## 最佳实践

### ✅ Do's

1. **始终使用混合检索**
   ```python
   # 推荐：并行混合 + RRF融合
   retriever = ParallelHybridRetriever(
       vector_retriever=vector_ret,
       keyword_retriever=bm25_ret,
       weights={'vector': 0.7, 'keyword': 0.3}
   )
   ```

2. **优先使用RRF融合**
   - 无需分数归一化
   - 鲁棒性强
   - 业界标准

3. **扩大初检召回数量**
   ```python
   # 初检召回2倍于最终需要的数量
   vector_results = vector_ret.search(query, top_k=20)
   keyword_results = bm25_ret.search(query, top_k=20)
   # 然后融合到Top-10
   ```

4. **根据场景调整权重**
   - 通用场景：vector=0.7, keyword=0.3
   - 专有名词多：vector=0.5, keyword=0.5
   - 概念性查询：vector=0.85, keyword=0.15

5. **实施动态权重调整**
   ```python
   adjuster = DynamicWeightAdjuster()
   weights = adjuster.adjust_weights(query)
   ```

6. **监控各路检索的效果**
   - 记录每路检索的召回情况
   - 分析哪些查询依赖向量检索
   - 分析哪些查询依赖关键词检索

7. **建立A/B测试框架**
   - 对比不同融合策略
   - 对比不同权重配置
   - 基于用户反馈优化

### ❌ Don'ts

1. **不要只使用单一检索方式**
   ```python
   # 错误！
   results = vector_only_search(query)
   
   # 正确：混合检索
   results = hybrid_search(query)
   ```

2. **不要忘记去重**

   ```python
   # 融合前确保去除重复文档
   ```


## 常见问题

### Q1: 应该选择并行还是串行混合检索？

**A**: 
- **并行**（推荐）：充分利用各路优势，召回率高
- **串行**：适合向量检索召回率高但精度低的场景

大多数场景建议使用并行混合。

### Q2: RRF的k值如何选择？

**A**: 
- 默认k=60（经验值）
- k越小，越重视排名靠前的结果
- k越大，排名差异越平缓
- 可以通过A/B测试找到最优值

### Q3: 向量检索和关键词检索的权重如何设置？

**A**: 
- 通用场景：0.7 / 0.3
- 技术文档：0.6 / 0.4
- 电商搜索：0.5 / 0.5
- 学术检索：0.8 / 0.2

建议从0.7/0.3开始，然后根据实际效果调整。

### Q4: Elasticsearch是否必须？

**A**: 
不是必须的。选择建议：
- 小规模（<100万文档）：FAISS + BM25即可
- 中等规模（100万-1000万）：Milvus/Qdrant
- 大规模（>1000万）或需要复杂过滤：Elasticsearch

### Q5: 混合检索会增加多少延迟？

**A**: 
- 并行混合：增加约50-100ms（两路检索并行执行）
- RRF融合：< 10ms
- 总体延迟增加可接受，但召回质量显著提升

优化方法：
- 并行执行各路检索
- 缓存热门查询结果
- 使用更快的向量索引（HNSW）

### Q6: 如何处理三路或更多路检索？

**A**: 
RRF天然支持多路融合：

```python
fused = reciprocal_rank_fusion(
    result_lists=[vector_results, bm25_results, es_results],
    weights=[0.5, 0.3, 0.2],
    top_k=10,
    k=60
)
```

只需添加更多检索路并设置相应权重即可。
