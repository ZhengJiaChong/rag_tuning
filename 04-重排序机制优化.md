## 概述

### 为什么需要重排序？

```
检索流程：
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  初检(召回)   │────→│  重排序       │────→│  LLM生成     │
│  100-500条   │     │  精选10-20条  │     │  最终答案    │
└──────────────┘     └──────────────┘     └──────────────┘
```

**重排序的价值**：
- ✅ **修正初检误差**：向量检索可能召回语义相关但不精确的结果
- ✅ **细粒度匹配**：考虑查询和文档的交互信号
- ✅ **多特征融合**：结合时效性、权威性等多维度信息
- ✅ **显著提升Top-K质量**：Top-3准确率可提升30-50%

### 重排序 vs 初检

| 特性 | 初检（Bi-Encoder） | 重排序（Cross-Encoder） |
|------|-------------------|------------------------|
| **速度** | 快（预计算向量） | 慢（实时计算） |
| **规模** | 百万级文档 | 百级候选集 |
| **精度** | 中等 | 高 |
| **适用** | 大规模召回 | 精细排序 |
| **模型** | 双塔结构 | 交互结构 |

## Cross-Encoder重排序

### 原理

Cross-Encoder将查询和文档拼接后一起输入模型，能够捕捉细粒度的token级别交互。

```
Bi-Encoder (初检):
Query → [Encoder] → Vector_Q
Doc   → [Encoder] → Vector_D
相似度 = cosine(Vector_Q, Vector_D)

Cross-Encoder (重排序):
[Query + Doc] → [Encoder] → 相关性分数
```

### 实现方案

#### 1. 使用BGE Reranker（推荐）

```python
from sentence_transformers import CrossEncoder
from typing import List, Dict

class CrossEncoderReranker:
    """
    基于Cross-Encoder的重排序器
    
    推荐使用BGE Reranker系列模型
    """
    
    def __init__(self, model_name='BAAI/bge-reranker-v2-m3'):
        """
        初始化重排序器
        
        Args:
            model_name: 模型名称
                - 'BAAI/bge-reranker-v2-m3': 多语言，性能最优
                - 'BAAI/bge-reranker-base': 速度快
                - 'BAAI/bge-reranker-large': 精度高
        """
        self.reranker = CrossEncoder(model_name)
        self.model_name = model_name
    
    def rerank(self, query: str, documents: List[Dict],
              top_k: int = 10) -> List[Dict]:
        """
        对候选文档进行重排序
        
        Args:
            query: 用户查询
            documents: 初检返回的候选文档列表
                       每个文档包含: {'id', 'content', 'score', ...}
            top_k: 返回的重排序后文档数量
        
        Returns:
            重排序后的文档列表（按相关性降序）
        """
        if not documents:
            return []
        
        # 构造输入对
        pairs = [[query, doc['content']] for doc in documents]
        
        # 批量预测相关性分数
        scores = self.reranker.predict(pairs, batch_size=32)
        
        # 附加分数到文档
        for i, doc in enumerate(documents):
            doc['rerank_score'] = float(scores[i])
            doc['rerank_model'] = self.model_name
        
        # 按重排序分数降序排列
        reranked = sorted(
            documents,
            key=lambda x: x['rerank_score'],
            reverse=True
        )
        
        return reranked[:top_k]
    
    def rerank_with_threshold(self, query: str, documents: List[Dict],
                             threshold: float = 0.0,
                             top_k: int = 10) -> List[Dict]:
        """
        带阈值过滤的重排序
        
        Args:
            threshold: 最低相关性分数阈值
        """
        reranked = self.rerank(query, documents, top_k=len(documents))
        
        # 过滤低分结果
        filtered = [doc for doc in reranked if doc['rerank_score'] >= threshold]
        
        return filtered[:top_k]


# 使用示例
if __name__ == '__main__':
    # 初始化
    reranker = CrossEncoderReranker('BAAI/bge-reranker-v2-m3')
    
    # 模拟初检结果
    query = "如何重置密码"
    candidates = [
        {'id': 'doc_1', 'content': '密码重置步骤...', 'score': 0.85},
        {'id': 'doc_2', 'content': '账户注册流程...', 'score': 0.82},
        {'id': 'doc_3', 'content': '忘记密码怎么办...', 'score': 0.78},
        # ... 更多候选
    ]
    
    # 重排序
    reranked = reranker.rerank(query, candidates, top_k=5)
    
    for i, doc in enumerate(reranked):
        print(f"{i+1}. [{doc['rerank_score']:.4f}] {doc['content'][:50]}...")
```

#### 3. 性能优化技巧

**批处理加速**：

```python
def batch_rerank(reranker, queries: List[str],
                document_lists: List[List[Dict]],
                batch_size: int = 32) -> List[List[Dict]]:
    """
    批量重排序
    
    适用于多个查询同时处理的场景
    """
    all_pairs = []
    query_doc_mapping = []
    
    # 收集所有需要重排序的(query, doc)对
    for query_idx, (query, docs) in enumerate(zip(queries, document_lists)):
        for doc_idx, doc in enumerate(docs):
            all_pairs.append([query, doc['content']])
            query_doc_mapping.append((query_idx, doc_idx))
    
    # 批量预测
    all_scores = reranker.reranker.predict(all_pairs, batch_size=batch_size)
    
    # 重新组织结果
    results = [[] for _ in range(len(queries))]
    
    for (query_idx, doc_idx), score in zip(query_doc_mapping, all_scores):
        doc = document_lists[query_idx][doc_idx].copy()
        doc['rerank_score'] = float(score)
        results[query_idx].append(doc)
    
    # 对每个查询的结果排序
    for query_results in results:
        query_results.sort(key=lambda x: x['rerank_score'], reverse=True)
    
    return results
```

**缓存热门查询**：

```python
import hashlib
import redis
import pickle

class CachedReranker:
    """带缓存的重排序器"""
    
    def __init__(self, base_reranker, redis_host='localhost', 
                 redis_port=6379, ttl=3600):
        self.base_reranker = base_reranker
        self.redis = redis.Redis(host=redis_host, port=redis_port, db=0)
        self.ttl = ttl  # 缓存过期时间（秒）
    
    def _get_cache_key(self, query: str, doc_ids: List[str]) -> str:
        """生成缓存key"""
        # 对查询和文档ID列表哈希
        content = f"{query}:{'|'.join(sorted(doc_ids))}"
        hash_obj = hashlib.md5(content.encode('utf-8'))
        return f"rerank:{hash_obj.hexdigest()}"
    
    def rerank(self, query: str, documents: List[Dict],
              top_k: int = 10) -> List[Dict]:
        """带缓存的重排序"""
        doc_ids = [doc['id'] for doc in documents]
        cache_key = self._get_cache_key(query, doc_ids)
        
        # 尝试从缓存获取
        cached = self.redis.get(cache_key)
        if cached:
            reranked = pickle.loads(cached)
            return reranked[:top_k]
        
        # 缓存未命中，执行重排序
        reranked = self.base_reranker.rerank(query, documents, top_k=len(documents))
        
        # 写入缓存
        serialized = pickle.dumps(reranked)
        self.redis.setex(cache_key, self.ttl, serialized)
        
        return reranked[:top_k]
```

## 多阶段重排序

### 两阶段重排序架构

结合速度和精度，先用快速模型筛选，再用高精度模型精细排序。

```
初检结果 (100条)
    ↓
┌─────────────────────┐
│ 第一阶段：快速筛选    │  BGE-Reranker-Base
│ 100条 → 30条        │  速度快，精度中等
└─────────────────────┘
    ↓
┌─────────────────────┐
│ 第二阶段：精细排序    │  BGE-Reranker-v2-M3
│ 30条 → 10条         │  速度慢，精度高
└─────────────────────┘
    ↓
  最终结果 (10条)
```

**实现代码**：

```python
class TwoStageReranker:
    """
    两阶段重排序器
    
    平衡速度和精度
    """
    
    def __init__(self, 
                 stage1_model='BAAI/bge-reranker-base',
                 stage2_model='BAAI/bge-reranker-v2-m3'):
        """
        初始化两阶段重排序器
        
        Args:
            stage1_model: 第一阶段模型（快速）
            stage2_model: 第二阶段模型（高精度）
        """
        self.stage1 = CrossEncoder(stage1_model)
        self.stage2 = CrossEncoder(stage2_model)
        
        # 各阶段的候选数量
        self.stage1_top_k = 30
        self.stage2_top_k = 10
    
    def rerank(self, query: str, documents: List[Dict],
              top_k: int = 10) -> List[Dict]:
        """
        两阶段重排序
        
        Args:
            query: 用户查询
            documents: 初检候选文档（建议50-100条）
            top_k: 最终返回数量
        
        Returns:
            重排序后的Top-K文档
        """
        if len(documents) <= top_k:
            # 候选数已很少，直接用高精度模型
            return self._single_stage_rerank(
                query, documents, self.stage2, top_k
            )
        
        # 第一阶段：快速筛选
        stage1_results = self._single_stage_rerank(
            query,
            documents,
            self.stage1,
            self.stage1_top_k
        )
        
        # 第二阶段：精细排序
        final_results = self._single_stage_rerank(
            query,
            stage1_results,
            self.stage2,
            top_k
        )
        
        return final_results
    
    def _single_stage_rerank(self, query: str, documents: List[Dict],
                            model: CrossEncoder, top_k: int) -> List[Dict]:
        """单阶段重排序"""
        pairs = [[query, doc['content']] for doc in documents]
        scores = model.predict(pairs, batch_size=32)
        
        for i, doc in enumerate(documents):
            doc['rerank_score'] = float(scores[i])
        
        # 排序
        reranked = sorted(
            documents,
            key=lambda x: x['rerank_score'],
            reverse=True
        )
        
        return reranked[:top_k]


# 使用示例
reranker = TwoStageReranker(
    stage1_model='BAAI/bge-reranker-base',
    stage2_model='BAAI/bge-reranker-v2-m3'
)

final_results = reranker.rerank(query, initial_candidates, top_k=10)
```

**性能对比**：

| 方案 | 延迟（100条候选） | Top-3准确率 | 成本 |
|------|------------------|------------|------|
| 仅Base模型 | ~200ms | 75% | 低 |
| 仅M3模型 | ~800ms | 85% | 高 |
| 两阶段 | ~350ms | 83% | 中 |

**优势**：
- ✅ 速度提升50%+（相比只用高精度模型）
- ✅ 精度损失<3%（相比只用高精度模型）
- ✅ 成本效益最优

## 特征工程重排序

除了使用深度学习模型，还可以结合多种手工特征进行重排序。

### 多特征融合框架

```python
import jieba
import math
from datetime import datetime
from typing import List, Dict

class FeatureBasedReranker:
    """
    基于多特征的重排序器
    
    结合语义、关键词、时效性、权威性等多种特征
    """
    
    def __init__(self, weights: Dict[str, float] = None):
        """
        初始化特征权重
        
        Args:
            weights: 各特征的权重
        """
        self.weights = weights or {
            'semantic_similarity': 0.35,   # 语义相似度
            'keyword_match': 0.25,         # 关键词匹配
            'freshness': 0.15,             # 新鲜度
            'authority': 0.10,             # 权威性
            'length_penalty': 0.10,        # 长度惩罚
            'position_boost': 0.05         # 位置提升
        }
        
        # 验证权重和为1
        assert abs(sum(self.weights.values()) - 1.0) < 0.01, \
            "权重之和必须为1"
    
    def rerank(self, query: str, documents: List[Dict],
              top_k: int = 10) -> List[Dict]:
        """
        基于多特征重排序
        
        Args:
            query: 用户查询
            documents: 候选文档列表，每个文档应包含:
                      - 'content': 文档内容
                      - 'vector_score': 向量相似度分数
                      - 'metadata': 元数据（可选）
        
        Returns:
            重排序后的文档列表
        """
        scored_docs = []
        
        for doc in documents:
            # 计算各项特征
            features = self.compute_features(query, doc)
            
            # 加权求和
            total_score = sum(
                features[feature_name] * self.weights[feature_name]
                for feature_name in self.weights
            )
            
            # 附加分数和特征详情
            doc['final_score'] = total_score
            doc['features'] = features
            
            scored_docs.append(doc)
        
        # 按总分排序
        scored_docs.sort(key=lambda x: x['final_score'], reverse=True)
        
        return scored_docs[:top_k]
    
    def compute_features(self, query: str, doc: Dict) -> Dict[str, float]:
        """
        计算各项特征分数
        
        Returns:
            特征字典，每个特征值在[0, 1]区间
        """
        features = {}
        
        # 1. 语义相似度（已有向量分数）
        features['semantic_similarity'] = doc.get('vector_score', 0.5)
        
        # 2. 关键词匹配度
        features['keyword_match'] = self.compute_keyword_overlap(
            query, doc['content']
        )
        
        # 3. 新鲜度分数
        features['freshness'] = self.compute_freshness(
            doc.get('metadata', {}).get('created_at')
        )
        
        # 4. 权威性分数
        features['authority'] = self.compute_authority(
            doc.get('metadata', {}).get('source')
        )
        
        # 5. 长度惩罚
        features['length_penalty'] = self.compute_length_penalty(
            len(doc['content']),
            ideal_length=300
        )
        
        # 6. 位置提升（初检排名）
        features['position_boost'] = self.compute_position_boost(
            doc.get('initial_rank', 999)
        )
        
        return features
    
    def compute_keyword_overlap(self, query: str, content: str) -> float:
        """
        计算关键词重叠度
        
        使用Jaccard相似度
        """
        # 分词
        query_words = set(jieba.lcut(query))
        content_words = set(jieba.lcut(content))
        
        # 去除停用词
        stop_words = {'的', '了', '在', '是', '我', '有', '和', '就', '不', '人'}
        query_words -= stop_words
        content_words -= stop_words
        
        if not query_words:
            return 0.0
        
        # Jaccard相似度
        intersection = query_words & content_words
        union = query_words | content_words
        
        if not union:
            return 0.0
        
        return len(intersection) / len(union)
    
    def compute_freshness(self, created_at: str) -> float:
        """
        计算新鲜度分数
        
        越近的文档分数越高，指数衰减
        
        Args:
            created_at: ISO格式日期字符串，如 '2024-01-15'
        
        Returns:
            新鲜度分数 [0, 1]
        """
        if not created_at:
            return 0.5  # 无日期信息，给中等分数
        
        try:
            doc_date = datetime.fromisoformat(created_at)
            days_diff = (datetime.now() - doc_date).days
            
            # 指数衰减：30天内为1，365天后接近0
            freshness = math.exp(-days_diff / 180)
            
            return max(0.0, min(1.0, freshness))
        
        except (ValueError, TypeError):
            return 0.5
    
    def compute_authority(self, source: str) -> float:
        """
        计算来源权威性分数
        
        Args:
            source: 文档来源
        
        Returns:
            权威性分数 [0, 1]
        """
        if not source:
            return 0.5
        
        # 权威性映射表
        authority_map = {
            'official_docs': 1.0,          # 官方文档
            'technical_manual': 0.9,       # 技术手册
            'api_reference': 0.85,         # API参考
            'tutorial': 0.7,               # 教程
            'blog_post': 0.6,              # 博客文章
            'user_forum': 0.4,             # 用户论坛
            'social_media': 0.3,           # 社交媒体
        }
        
        # 简化的来源分类
        source_lower = source.lower()
        
        if 'official' in source_lower or 'docs' in source_lower:
            return authority_map['official_docs']
        elif 'manual' in source_lower:
            return authority_map['technical_manual']
        elif 'api' in source_lower:
            return authority_map['api_reference']
        elif 'tutorial' in source_lower:
            return authority_map['tutorial']
        elif 'blog' in source_lower:
            return authority_map['blog_post']
        elif 'forum' in source_lower:
            return authority_map['user_forum']
        else:
            return 0.5  # 未知来源
    
    def compute_length_penalty(self, actual_length: int,
                              ideal_length: int = 300) -> float:
        """
        长度惩罚
        
        偏离理想长度则扣分，避免过长或过短的文档
        
        Args:
            actual_length: 实际长度（字符数）
            ideal_length: 理想长度
        
        Returns:
            长度惩罚分数 [0, 1]
        """
        if actual_length == 0:
            return 0.0
        
        ratio = actual_length / ideal_length
        
        # 在0.5-2.0倍理想长度范围内，给满分
        if 0.5 <= ratio <= 2.0:
            return 1.0
        
        # 超出范围，线性递减
        penalty = max(0.1, 1.0 - abs(ratio - 1.0))
        
        return penalty
    
    def compute_position_boost(self, initial_rank: int) -> float:
        """
        位置提升
        
        初检排名靠前的文档给予轻微提升
        
        Args:
            initial_rank: 初检排名（从1开始）
        
        Returns:
            位置提升分数 [0, 1]
        """
        if initial_rank <= 0:
            return 0.5
        
        # 前10名给较高分数，之后递减
        if initial_rank <= 10:
            return 1.0 - (initial_rank - 1) * 0.05
        else:
            return max(0.1, 0.5 - (initial_rank - 10) * 0.02)
```

### 特征重要性分析

```python
class FeatureImportanceAnalyzer:
    """特征重要性分析器"""
    
    def __init__(self, reranker: FeatureBasedReranker):
        self.reranker = reranker
    
    def analyze(self, test_cases: List[Dict]) -> Dict[str, float]:
        """
        分析各特征的重要性
        
        Args:
            test_cases: 测试用例列表
                       [{'query': ..., 'documents': ..., 'relevant_ids': ...}, ...]
        
        Returns:
            各特征的重要性分数
        """
        feature_impact = {feat: 0.0 for feat in self.reranker.weights}
        
        for case in test_cases:
            query = case['query']
            documents = case['documents']
            relevant_ids = set(case['relevant_ids'])
            
            # 原始重排序结果
            original_results = self.reranker.rerank(query, documents, top_k=10)
            original_ndcg = self.compute_ndcg(original_results, relevant_ids)
            
            # 逐个特征消融实验
            for feature in feature_impact.keys():
                # 临时将该特征权重设为0
                original_weight = self.reranker.weights[feature]
                self.reranker.weights[feature] = 0.0
                
                # 重新归一化其他权重
                remaining_weight = 1.0 - original_weight
                for other_feat in self.reranker.weights:
                    if other_feat != feature:
                        self.reranker.weights[other_feat] /= remaining_weight
                
                # 重新重排序
                ablated_results = self.reranker.rerank(query, documents, top_k=10)
                ablated_ndcg = self.compute_ndcg(ablated_results, relevant_ids)
                
                # 记录NDCG下降
                feature_impact[feature] += (original_ndcg - ablated_ndcg)
                
                # 恢复权重
                self.reranker.weights = {
                    feat: w / sum(self.reranker.weights.values())
                    for feat, w in self.reranker.weights.items()
                }
        
        # 平均影响
        num_cases = len(test_cases)
        for feature in feature_impact:
            feature_impact[feature] /= num_cases
        
        return feature_impact
    
    def compute_ndcg(self, results: List[Dict], relevant_ids: set) -> float:
        """计算NDCG@10"""
        dcg = 0.0
        for i, doc in enumerate(results[:10]):
            if doc['id'] in relevant_ids:
                relevance = 1.0
                dcg += relevance / math.log2(i + 2)
        
        # 理想DCG
        ideal_relevant = min(len(relevant_ids), 10)
        idcg = sum(1.0 / math.log2(i + 2) for i in range(ideal_relevant))
        
        return dcg / idcg if idcg > 0 else 0.0
```

## 多样性控制

避免返回过于相似的结果，提供多样化的信息。

### MMR（Maximal Marginal Relevance）

```python
import numpy as np
from sklearn.metrics.pairwise import cosine_similarity
from typing import List, Dict

def mmr_rerank(query_embedding: np.ndarray,
              documents: List[Dict],
              embeddings: np.ndarray,
              lambda_param: float = 0.7,
              top_k: int = 10) -> List[Dict]:
    """
    MMR重排序：平衡相关性和多样性
    
    公式：MMR = argmax [λ·Rel(Q,D) - (1-λ)·max Sim(D, D_selected)]
    
    Args:
        query_embedding: 查询向量
        documents: 候选文档列表
        embeddings: 文档向量矩阵 (n_docs, dim)
        lambda_param: 相关性权重（1-多样性权重）
                     0.7表示70%相关性，30%多样性
        top_k: 返回数量
    
    Returns:
        多样化排序后的文档列表
    """
    if not documents:
        return []
    
    selected_indices = []
    remaining_indices = list(range(len(documents)))
    
    while len(selected_indices) < top_k and remaining_indices:
        best_idx = None
        best_score = -float('inf')
        
        for idx in remaining_indices:
            # 1. 与查询的相关性
            relevance = cosine_similarity(
                query_embedding.reshape(1, -1),
                embeddings[idx].reshape(1, -1)
            )[0][0]
            
            # 2. 与已选文档的最大相似度（冗余度）
            redundancy = 0.0
            if selected_indices:
                similarities = cosine_similarity(
                    embeddings[idx].reshape(1, -1),
                    embeddings[selected_indices]
                )[0]
                redundancy = np.max(similarities)
            
            # 3. MMR分数
            mmr_score = lambda_param * relevance - (1 - lambda_param) * redundancy
            
            if mmr_score > best_score:
                best_score = mmr_score
                best_idx = idx
        
        if best_idx is not None:
            selected_indices.append(best_idx)
            remaining_indices.remove(best_idx)
    
    # 返回选中的文档
    return [documents[i] for i in selected_indices]


# 使用示例
def diverse_search(query, initial_results, embeddings, top_k=10):
    """多样化搜索"""
    query_emb = embedding_model.encode([query])[0]
    
    doc_embeddings = np.array([
        embedding_model.encode([doc['content']])[0]
        for doc in initial_results

        query_embedding=query_emb,

```

| 0.7 | 70% | 30% | 通用场景（默认） |

### 聚类去重

```python
from sklearn.cluster import KMeans

def cluster_based_diversity(documents: List[Dict],
                           embeddings: np.ndarray,
                           top_k: int = 10,
                           n_clusters: int = 5) -> List[Dict]:
    """
    基于聚类的多样性选择
    
    策略：
    1. 对候选文档聚类
    2. 从每个簇中选择最相关的文档
    3. 确保覆盖不同主题
    """
    if len(documents) <= top_k:
        return documents
    
    # 聚类
    kmeans = KMeans(n_clusters=n_clusters, random_state=42)
    cluster_labels = kmeans.fit_predict(embeddings)
    
    # 按簇分组
    clusters = {}
    for idx, label in enumerate(cluster_labels):
        if label not in clusters:
            clusters[label] = []
        clusters[label].append(idx)
    
    # 从每个簇中选择文档
    selected = []
    docs_per_cluster = top_k // n_clusters
    remainder = top_k % n_clusters
    
    for cluster_id, indices in clusters.items():
        # 该簇应选择的文档数
        n_select = docs_per_cluster + (1 if cluster_id < remainder else 0)
        
        # 选择该簇中最相关的n_select个文档（假设已按相关性排序）
        selected.extend(indices[:n_select])
    
    # 返回选中的文档
    return [documents[i] for i in selected]
```

## 个性化重排序

根据用户画像和历史行为调整排序。

### 用户画像驱动的重排序

```python
class PersonalizedReranker:
    """
    个性化重排序器
    
    基于用户兴趣和历史行为调整排序
    """
    
    def __init__(self, base_reranker):
        self.base_reranker = base_reranker
    
    def rerank(self, query: str, documents: List[Dict],
              user_profile: Dict, top_k: int = 10) -> List[Dict]:
        """
        个性化重排序
        
        Args:
            query: 用户查询
            documents: 候选文档
            user_profile: 用户画像
                         {
                             'interests': ['机器学习', 'Python'],
                             'expertise_level': 'intermediate',
                             'history': [...]
                         }
            top_k: 返回数量
        """
        # 先进行基础重排序
        reranked = self.base_reranker.rerank(query, documents, top_k=len(documents))
        
        # 个性化调整
        personalized_results = []
        
        for doc in reranked:
            base_score = doc.get('rerank_score', doc.get('final_score', 0.5))
            
            # 1. 兴趣匹配度
            interest_score = self.compute_interest_match(
                user_profile.get('interests', []),
                doc.get('metadata', {}).get('tags', [])
            )
            
            # 2. 专业 level 匹配
            expertise_score = self.compute_expertise_match(
                user_profile.get('expertise_level', 'intermediate'),
                doc.get('metadata', {}).get('difficulty', 'medium')
            )
            
            # 3. 历史行为偏好
            behavior_score = self.compute_behavior_score(
                user_profile.get('history', []),
                doc.get('metadata', {}).get('source')
            )
            
            # 综合分数
            personalized_score = (
                0.6 * base_score +
                0.25 * interest_score +
                0.10 * expertise_score +
                0.05 * behavior_score
            )
            
            doc['personalized_score'] = personalized_score
            personalized_results.append(doc)
        
        # 按个性化分数排序
        personalized_results.sort(
            key=lambda x: x['personalized_score'],
            reverse=True
        )
        
        return personalized_results[:top_k]
    
    def compute_interest_match(self, user_interests: List[str],
                              doc_tags: List[str]) -> float:
        """计算兴趣匹配度"""
        if not user_interests or not doc_tags:
            return 0.5
        
        user_set = set(user_interests)
        doc_set = set(doc_tags)
        
        # Jaccard相似度
        intersection = user_set & doc_set
        union = user_set | doc_set
        
        if not union:
            return 0.5
        
        return len(intersection) / len(union)
    
    def compute_expertise_match(self, user_level: str,
                               doc_difficulty: str) -> float:
        """计算专业水平匹配度"""
        level_map = {
            'beginner': 1,
            'intermediate': 2,
            'advanced': 3
        }
        
        user_val = level_map.get(user_level, 2)
        doc_val = level_map.get(doc_difficulty, 2)
        
        # 差距越小分数越高
        diff = abs(user_val - doc_val)
        
        if diff == 0:
            return 1.0
        elif diff == 1:
            return 0.7
        else:
            return 0.3
    
    def compute_behavior_score(self, history: List[Dict],
                              source: str) -> float:
        """计算历史行为偏好"""
        if not history:
            return 0.5
        
        # 统计用户点击的来源分布
        source_clicks = {}
        for interaction in history[-50:]:  # 最近50次
            src = interaction.get('source', '')
            if interaction.get('clicked', False):
                source_clicks[src] = source_clicks.get(src, 0) + 1
        
        total_clicks = sum(source_clicks.values())
        if total_clicks == 0:
            return 0.5
        
        # 该来源的点击率
        click_rate = source_clicks.get(source, 0) / total_clicks
        
        return min(1.0, click_rate * 2)  # 放大差异
```

## 评估与优化

### 重排序效果评估

```python
class RerankingEvaluator:
    """重排序效果评估器"""
    
    def __init__(self):
        pass
    
    def evaluate_ndcg(self, reranker, test_cases: List[Dict],
                     k_values: List[int] = [1, 3, 5, 10]) -> Dict:
        """
        评估NDCG指标
        
        Args:
            test_cases: [{'query': ..., 'candidates': ..., 'relevance': {...}}, ...]
                       relevance: {doc_id: relevance_score}
        """
        ndcg_scores = {k: [] for k in k_values}
        
        for case in test_cases:
            query = case['query']
            candidates = case['candidates']
            relevance = case['relevance']
            
            # 重排序
            reranked = reranker.rerank(query, candidates, top_k=max(k_values))
            
            # 计算各K值的NDCG
            for k in k_values:
                ndcg = self.compute_ndcg_at_k(reranked[:k], relevance, k)
                ndcg_scores[k].append(ndcg)
        
        # 汇总
        results = {}
        for k in k_values:
            results[f'NDCG@{k}'] = np.mean(ndcg_scores[k])
        
        return results
    
    def compute_ndcg_at_k(self, results: List[Dict],
                         relevance: Dict[str, float],
                         k: int) -> float:
        """计算NDCG@K"""
        dcg = 0.0
        
        for i, doc in enumerate(results[:k]):
            rel = relevance.get(doc['id'], 0.0)
            dcg += rel / math.log2(i + 2)
        
        # 理想DCG
        ideal_rels = sorted(relevance.values(), reverse=True)[:k]
        idcg = sum(rel / math.log2(i + 2) for i, rel in enumerate(ideal_rels))
        
        return dcg / idcg if idcg > 0 else 0.0
    
    def evaluate_mrr(self, reranker, test_cases: List[Dict]) -> float:
        """评估MRR（Mean Reciprocal Rank）"""
        mrr_scores = []
        
        for case in test_cases:
            query = case['query']
            candidates = case['candidates']
            relevant_ids = case['relevant_ids']  # 相关文档ID列表
            
            # 重排序
            reranked = reranker.rerank(query, candidates, top_k=20)
            
            # 找到第一个相关文档的排名
            for rank, doc in enumerate(reranked, 1):
                if doc['id'] in relevant_ids:
                    mrr_scores.append(1.0 / rank)
                    break
            else:
                mrr_scores.append(0.0)
        
        return np.mean(mrr_scores)
    
    def compare_before_after(self, reranker, test_cases: List[Dict]) -> Dict:
        """对比重排序前后的效果"""
        before_metrics = {
            'ndcg@1': [],
            'ndcg@3': [],
            'ndcg@5': [],
            'mrr': []
        }
        
        after_metrics = {
            'ndcg@1': [],
            'ndcg@3': [],
            'ndcg@5': [],
            'mrr': []
        }
        
        for case in test_cases:
            query = case['query']
            candidates = case['candidates']
            relevance = case['relevance']
            relevant_ids = case['relevant_ids']
            
            # 重排序前（按初始分数排序）
            before_sorted = sorted(
                candidates,
                key=lambda x: x.get('score', 0),
                reverse=True
            )
            
            # 重排序后
            after_sorted = reranker.rerank(query, candidates, top_k=20)
            
            # 计算指标
            for metrics, results in [(before_metrics, before_sorted),
                                     (after_metrics, after_sorted)]:
                metrics['ndcg@1'].append(
                    self.compute_ndcg_at_k(results[:1], relevance, 1)
                )
                metrics['ndcg@3'].append(
                    self.compute_ndcg_at_k(results[:3], relevance, 3)
                )
                metrics['ndcg@5'].append(
                    self.compute_ndcg_at_k(results[:5], relevance, 5)
                )
                
                # MRR
                for rank, doc in enumerate(results, 1):
                    if doc['id'] in relevant_ids:
                        metrics['mrr'].append(1.0 / rank)
                        break
                else:
                    metrics['mrr'].append(0.0)
        
        # 汇总
        summary = {}
        for metric in before_metrics:
            before_avg = np.mean(before_metrics[metric])
            after_avg = np.mean(after_metrics[metric])
            improvement = (after_avg - before_avg) / before_avg * 100
            
            summary[metric] = {
                'before': before_avg,
                'after': after_avg,
                'improvement_percent': improvement
            }
        
        return summary


# 使用示例
evaluator = RerankingEvaluator()

# 准备测试数据
test_cases = [...]

# 评估
results = evaluator.evaluate_ndcg(reranker, test_cases)
print("NDCG指标:", results)

# 对比前后效果
comparison = evaluator.compare_before_after(reranker, test_cases)
for metric, values in comparison.items():
    print(f"{metric}:")
    print(f"  重排序前: {values['before']:.4f}")
    print(f"  重排序后: {values['after']:.4f}")
    print(f"  提升: {values['improvement_percent']:.2f}%")
```

## 最佳实践

### ✅ Do's

1. **始终使用重排序**
   - 即使只有20-30个候选，重排序也能提升10-20%精度
   - 优先级：重排序 > 混合检索 > Query改写

2. **选择合适的模型**
   ```python
   # 通用场景
   reranker = CrossEncoderReranker('BAAI/bge-reranker-v2-m3')
   
   # 延迟敏感
   reranker = CrossEncoderReranker('BAAI/bge-reranker-base')
   
   # 两阶段（推荐）
   reranker = TwoStageReranker()
   ```

3. **实施批处理和缓存**
   ```python
   # 批处理
   scores = reranker.predict(pairs, batch_size=32)
   
   # 缓存
   cached_reranker = CachedReranker(base_reranker, ttl=3600)
   ```

4. **监控重排序延迟**
   - Base模型：< 200ms（100条候选）
   - M3模型：< 800ms（100条候选）
   - 设置超时保护

5. **定期评估效果**
   - 每月运行一次NDCG评估
   - A/B测试不同模型
   - 收集用户反馈

6. **结合多样性控制**
   ```python
   # 探索性查询使用MMR
   if query_type == 'exploratory':
       results = mmr_rerank(..., lambda_param=0.5)
   ```

### ❌ Don'ts

1. **不要对所有候选重排序**
   ```python
   # 错误！候选太多会导致延迟过高
   reranker.rerank(query, all_1000_documents, top_k=10)
   
   # 正确：先筛选到50-100条
   candidates = initial_search(query, top_k=100)
   reranker.rerank(query, candidates, top_k=10)
   ```

2. **不要忘记归一化**
   ```python
   # 确保向量已归一化
   embeddings = normalize_embeddings(embeddings)
   ```

3. **不要硬编码权重**
   - 通过A/B测试优化权重
   - 根据业务场景调整

4. **不要忽略冷启动问题**
   - 新用户/新查询使用默认策略
   - 逐步积累个性化数据

5. **不要在CPU上处理大量请求**
   ```python
   # 生产环境使用GPU
   reranker = setup_gpu_reranker(model_name, gpu_id=0)
   ```

## 常见问题

### Q1: 重排序会增加多少延迟？

**A**:
- Base模型：~2ms/条（100条约200ms）
- M3模型：~8ms/条（100条约800ms）
- 两阶段：~3.5ms/条（100条约350ms）

通过GPU加速和批处理可降低30-50%。

### Q2: 是否需要为每种场景训练不同的重排序模型？

**A**: 
初期使用通用模型即可。当有以下情况时考虑微调：
- 有大量标注数据（>5000个query-doc对）
- 通用模型在特定领域表现不佳
- 有专门的团队维护ML模型

### Q3: 如何处理重排序失败或超时？

**A**:
```python
def safe_rerank(reranker, query, documents, timeout_ms=1000):
    """安全的重排序，带超时保护"""
    try:
        # 设置超时
        result = reranker.rerank(query, documents, top_k=10)
        return result
    except TimeoutError:
        # 超时，返回初检结果
        logging.warning(f"重排序超时，返回初检结果")
        return documents[:10]
    except Exception as e:
        # 其他错误，返回初检结果
        logging.error(f"重排序失败: {e}")
        return documents[:10]
```

### Q4: MMR的lambda参数如何选择？

**A**:
- 精确查询（产品型号、错误码）：lambda=0.9
- 通用问答：lambda=0.7（默认）
- 探索性查询：lambda=0.5
- 创意生成：lambda=0.3

可通过A/B测试找到最优值。

### Q5: 重排序是否总是有效？

**A**: 
大多数情况下有效，但以下场景效果有限：
- 初检质量已经很高（NDCG@10 > 0.9）
- 候选集太小（<10条）
- 查询和文档都非常短

在这些场景下，可以考虑跳过重排序以节省成本。
